Backed by the custom <CanvasToolbar> component (src/components/CanvasToolbar/CanvasToolbar.tsx). @radix-ui/react-toolbar exists but is NOT installed in this project — its ToggleGroup / Separator / focus-roving features would be useful if the toolbar grows, but the current zoom / fullscreen / version-chip surface is small enough that native <button>s with grouped <div> dividers are clearer to read.

// CanvasToolbar — composed inline at the viewer level
import { CanvasToolbar } from '@/components/CanvasToolbar';
import { VersionChip } from '@/components/VersionChip';

function MockupViewer({ zoom, setZoom, isFullscreen, versions, currentVersion, canvasRef }) {
  return (
    <CanvasToolbar
      boundsRef={canvasRef}                            // drag clamps to this rect (8 px margin)
      resetPositionKey={isFullscreen ? 'fs' : 'win'}    // reset offset on fullscreen flip
    >
      <CanvasToolbar.Group>                          // zoom controls
        <button className={styles.btn}
                data-tooltip="Zoom out (Ctrl+−)"
                aria-label="Zoom out"
                onClick={() => setZoom(zoom - 10)}
                disabled={zoom <= MIN_ZOOM}>
          <MinusIcon />
        </button>
        <button className={styles.zoom}
                onClick={() => setZoom(100)}
                aria-label={`Reset to 100% (current: ${zoom}%)`}>
          {zoom}%
        </button>
        <button className={styles.btn}
                data-tooltip="Zoom in (Ctrl++)"
                onClick={() => setZoom(zoom + 10)}
                disabled={zoom >= MAX_ZOOM}>
          <PlusIcon />
        </button>
      </CanvasToolbar.Group>

      <CanvasToolbar.Group>                          // fullscreen toggle
        <button className={cn(styles.btn, isFullscreen && styles.active)}
                data-tooltip={isFullscreen ? 'Exit fullscreen (F)' : 'Fullscreen (F)'}
                aria-pressed={isFullscreen}
                onClick={toggleFullscreen}>
          <FullscreenIcon />
        </button>
      </CanvasToolbar.Group>

      <CanvasToolbar.Group>                          // version chip (popover)
        <VersionChip versions={versions} current={currentVersion} />
      </CanvasToolbar.Group>

      <CanvasToolbar.DragHandle />                   // always last group — see annotations-rail
    </CanvasToolbar>
  );
}

Notes:

  • @radix-ui/react-toolbar is not installed. The current surface is small enough that native <button> grouped by <div className={styles.group}> is the right fit. Add Radix Toolbar only when we need roving focus / arrow-key navigation (current shipped controls are tab-focusable individually, which works for 4 items).
  • Buttons are native <button> with the .btn class. Radix has no Button primitive — pair every icon button with an aria-label and a data-tooltip.
  • Drag handle, zoom logic, and version chip are independent compounds. CanvasToolbar exposes Group + DragHandle slots; the actual zoom math (zoom.ts) and version popover (VersionChip) live in their own files.
  • boundsRef + resetPositionKey mirror the annotations-rail pattern. Toolbar drag clamps to the canvas bounds; the position resets when the bounds change (fullscreen toggle).
  • Toggle buttons use aria-pressed (not aria-checked — that's for checkbox/radio). The fullscreen button gets .active visual state and aria-pressed="true" simultaneously.
  • Zoom label is a button, not a span — clicking resets to 100%. Include the current value in aria-label so SR users know the state before activating.
State decision matrix
StateTriggerVisualAria
resting Default. No interaction. Glass surface, all buttons in --text-muted resting tone. Root: role="toolbar", aria-label="Canvas controls".
btn-hover :hover on any .btn. Background lifts to --surface-hover; icon colour to --text-bright. —
btn-active Toggle button is pressed (e.g. fullscreen on). --accent-soft bg + inset 1 px --accent-overlay-mid ring + --accent-bright icon. aria-pressed="true" on the button.
btn-disabled Zoom at min/max → corresponding zoom button disabled. 0.4 opacity, cursor:not-allowed. disabled attribute — SR-announced natively.
dragging mousedown on the grab handle, drag in progress. Toolbar gets extra 1 px --accent-overlay-mid ring on top of base shadow; grab icon brightens. —
version-popover-open Version chip clicked. Popover paints in top-layer above the toolbar. Chevron rotates 180°. Chip: aria-expanded="true", aria-haspopup="listbox".
"dragging" and "version-popover-open" are internal states — the consumer doesn't toggle them. btn-active is consumer-driven (e.g. fullscreen state lives in the parent).

Behavior

  • Drag-to-reposition. mousedown on the grab handle starts a drag. Position clamps to the canvas bounds with an 8 px margin on every edge.
  • Reset on bounds change. resetPositionKey change clears the drag offset — toolbar returns to center-bottom default.
  • Zoom controls. Out / in buttons step by 10% (configurable). Label is clickable and resets to 100%. Both side buttons disable at min/max.
  • Fullscreen toggle. Uses Fullscreen API on the canvas container; toolbar rides along inside. aria-pressed reflects state.
  • Version chip. Opens a popover (see popover) with the version list. Click outside or Escape closes; chevron rotates while open.
  • Tooltips on every button. Pair each icon with data-tooltip for the hint + aria-label for SR — see 16-tooltip.html.
  • Keyboard shortcuts. Ctrl+− / Ctrl++ for zoom, F for fullscreen. Registered globally by the viewer, not by individual buttons.
  • Reduced motion. Under prefers-reduced-motion: reduce, the btn hover/active transitions collapse to a 0 ms snap via the token override at the top of the file.
