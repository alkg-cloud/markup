07 — New Project Dialog

Project creation + edit dialog (src/components/NewProjectDialog/NewProjectDialog.tsx). Composes three primitives that already ship as their own DS entries: the dialog surface (@radix-ui/react-dialog via RadixDialog), input-field for the name with @radix-ui/react-form ValidityState validation, and icon-picker for the project glyph. URL-safe validation runs on every keystroke (via validateUrlSafeName); submit is disabled while the name is empty, invalid, or a request is in flight. Same component renders both Create and Edit modes — only the title, prefilled values, and primary button label differ. This is a single-field form; per DS 14 guidelines the close-X (showCloseButton) is optional and is omitted by default in favour of the Cancel button.

React API — composes RadixDialog + Form.Root + InputField + IconPicker

No new primitive — this is a feature dialog that composes four existing pieces. The RadixDialog compound (src/components/Dialog/RadixDialog.tsx) wraps @radix-ui/react-dialog with the project's glass surface. @radix-ui/react-form provides ValidityState-driven validation via [data-invalid]. InputField (DS 28) renders the labelled input with help/error rotation. IconPicker (DS 06) renders the glyph picker.

import * as Form from '@radix-ui/react-form';
import { RadixDialog } from '@/components/Dialog/RadixDialog';
import { InputField } from '@/components/InputField';
import { IconPicker } from '@/components/IconPicker/IconPicker';
import { URL_SAFE_NAME_PATTERN, validateUrlSafeName } from '@/lib/validation/url-safe-name';

// Bare pattern (no ^…$ anchors) — ValidityState anchors automatically.
const NAME_PATTERN = URL_SAFE_NAME_PATTERN.source.replace(/^\^|\$$/g, '');

<RadixDialog.Root open={open} onOpenChange={handleOpenChange}>
  <RadixDialog.Portal>
    <RadixDialog.Overlay />                            // scrim — blur(16px) + --scrim-glass-bg
    <RadixDialog.Content aria-describedby={undefined}> // no Description child → suppress warning
      <RadixDialog.Title>{isEdit ? 'Edit Project' : 'New Project'}</RadixDialog.Title>

      <Form.Root className={styles.form} onSubmit={handleFormSubmit}>
        <InputField.Root name="name" data-state={nameError ? 'error' : undefined}>
          <InputField.Label>Project name</InputField.Label>
          <InputField.Control asChild>                  // Slot — caller owns the <input>
            <input type="text" required pattern={NAME_PATTERN}
                   placeholder="My-Project"
                   value={name} onChange={(e) => setName(e.target.value)} />
          </InputField.Control>

          {nameError
            ? <InputField.Message forceMatch>{nameError.message}</InputField.Message>
            : <InputField.Help>Letters, digits, hyphens, or underscores.</InputField.Help>}

          <InputField.Message match="patternMismatch">
            Use letters, digits, hyphens, or underscores only.
          </InputField.Message>
          <InputField.Message match="valueMissing">
            A project name is required.
          </InputField.Message>
        </InputField.Root>

        <div className={styles.iconField}>                // non-input field, no InputField wrapper
          <span className={styles.iconLabel}>Icon</span>
          <IconPicker value={icon} onSelect={setIcon} />
        </div>

        <div className={styles.actions}>
          <RadixDialog.Close asChild>
            <button type="button" className={styles.btnSecondary} disabled={loading}>
              Cancel
            </button>
          </RadixDialog.Close>
          <Form.Submit asChild>
            <button type="submit" className={styles.btnAccent}
                    disabled={!canSubmit || loading}>
              {isEdit ? 'Update' : 'Create'}
            </button>
          </Form.Submit>
        </div>
      </Form.Root>
    </RadixDialog.Content>
  </RadixDialog.Portal>
</RadixDialog.Root>

Notes:

  • Pattern source of truth. URL_SAFE_NAME_PATTERN lives in src/lib/validation/url-safe-name.ts. The native pattern attribute uses the bare form (no ^…$) because ValidityState anchors automatically.
  • Dual error path. Synchronous validator output (validateUrlSafeName(trimmedName)) is shown via InputField.Message forceMatch. Browser-driven patternMismatch / valueMissing fire when ValidityState catches an issue the validator missed (e.g. empty submit). Both render at --danger with the shake animation defined in input-field.module.css.
  • aria-describedby suppression (custom). The dialog has no RadixDialog.Description child (the help line lives inside the InputField), so aria-describedby={undefined} on Content silences Radix's dev warning. This is documented in DS 14.
  • Icon field is non-input. The picker isn't a <input> so it doesn't compose InputField.Root — it gets its own label/grid wrapper. Selection persists across tab switches; the consumer's icon state is the source of truth.
  • Loading lock. Both buttons get :disabled (0.5 opacity, not-allowed cursor) while the POST/PATCH is in flight; the dialog stays mounted until the request resolves. On success: close + toast. On failure: keep open + toast.
  • Auto-focus. RadixDialog auto-focuses the first focusable element inside Content on open — that's the name input. No autoFocus attribute (forbidden by Biome's a11y rule).
  • Same component, two modes. Pass project for edit mode; omit it for create mode. Title and button label switch automatically.

success (custom) POST / PATCH 2xx response. Dialog closes via handleClose; useToast().show() announces "Project created" / "Project updated"; onSaved(project) fires. Toast surfaces with role="status".
error (custom) POST / PATCH non-2xx response. Dialog stays open; toast shows "Failed to create/update project"; loading resets so the user can retry. Toast surfaces error message.

Behavior
  • Open lifecycle. When open flips true, a useEffect seeds name and icon from project (edit) or empties them (create). RadixDialog auto-focuses the name input.
  • Live validation. Every keystroke runs validateUrlSafeName(trimmedName) via useMemo. The result is the canonical synchronous error; ValidityState catches the rest on submit.
  • Submit gate. canSubmit = trimmedName.length > 0 && !nameError. The submit handler also guards on loading to prevent double-submits.
  • Network. POST /api/projects for create, PATCH /api/projects/:id for edit. Body is { name: trimmedName, icon }. On success: close, fire onSaved(saved), toast. On failure: stay open, toast.
  • Close paths. Cancel button (RadixDialog.Close asChild), Escape, scrim click — all funnel through handleOpenChange(false) → handleClose which resets local state before calling onClose.
  • Toast feedback. useToast().show() surfaces both success and failure outside the dialog — see 17-toast.
  • Composes three DS entries. Dialog surface (dialog) + name field (input-field) + glyph picker (icon-picker). Every visual decision in those entries cascades into this one.
  • Reduced motion. Dialog enter animation collapses to a 0 ms opacity flip via the token override.
