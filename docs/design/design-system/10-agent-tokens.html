10 — Agent Tokens

Settings page for managing API tokens used by non-browser agents. Implemented as src/app/(app)/settings/agents/AgentsClient.tsx behind /settings/agents (admin-only route, gated by middleware + entry in avatar-menu's Admin block). Provides: list of existing tokens (masked preview + relative timestamps), inline create form, post-create one-time reveal banner (the only moment the plaintext exists in the UI), and per-token copy + revoke actions. Revoke uses useConfirm for the destructive confirmation.

React API — custom page (admin route, no Radix primitive)

Page-level component. There is no Radix primitive for "list of tokens with create + revoke" — this is a domain-specific surface composed from the project's own building blocks: AppMain (page layout), native <form> for create, useConfirm (@radix-ui/react-alert-dialog) for revoke. The destructive confirmation is the only Radix primitive in the page.

// src/app/(app)/settings/agents/AgentsClient.tsx — abridged
import { AppMain } from '@/components/AppMain/AppMain';
import { useConfirm } from '@/components/ConfirmDialog';
import { VscAdd, VscCopy, VscKey, VscTrash } from 'react-icons/vsc';

const [tokens, setTokens] = useState<AgentToken[]>(initial);
const [revealed, setRevealed] = useState<CreatedToken | null>(null);
const { confirm, dialog: confirmDialog } = useConfirm();          // see 14-dialog

async function onCreate(e) {
  e.preventDefault();
  const res = await fetch('/api/agent-tokens', {
    method: 'POST', headers: { 'content-type': 'application/json' },
    body: JSON.stringify({ name }),
  });
  if (!res.ok) { setError(…); return; }
  const created = await res.json();
  setRevealed(created);                                              // one-time plaintext
  await refresh();
}

async function onRevoke(id, tokenName) {
  const ok = await confirm({
    title: 'Revoke agent token',
    description: `"${tokenName}" will lose API access immediately. …`,
    confirmLabel: 'Revoke',
    danger: true,                                              // → .btn-danger styling
  });
  if (!ok) return;
  await fetch(`/api/agent-tokens/${id}`, { method: 'DELETE' });
  await refresh();
}

return (
  <AppMain variant="centered" ariaLabel="Agent tokens settings">
    {confirmDialog}                                                  // mount once for the portal root
    <h1>Agent Tokens</h1>
    {revealed && <RevealBanner token={revealed} />}              // post-create one-time
    {showForm && <CreateForm onSubmit={onCreate} />}
    <TokenList tokens={tokens} onRevoke={onRevoke} />
  </AppMain>
);

Notes:

  • useConfirm is the Promise-based revoke confirmation. danger: true swaps the primary action to .btn-danger. confirmDialog must be mounted once inside AppMain so the portal has a tree root. See 14-dialog.
  • The reveal banner is intentionally not a dialog — it doesn't take focus, doesn't trap, doesn't dismiss on Escape. It's a persistent in-flow notice that survives until the user navigates away or creates another token. Using a dialog here would force a click-out for every token.
  • The page replaces the banned window.confirm with useConfirm (see docs/code-style.md § Never use native browser dialogs).
  • Native <form> with pattern="[A-Za-z0-9_-]+" + required attributes does the sync validation; the server is authoritative for uniqueness and length. A @radix-ui/react-form migration is on the table — see 28-input-field for the pattern.
  • Server route: src/app/api/agent-tokens/*. The list endpoint never returns plaintext — only {prefix, lastFour}. The create endpoint returns plaintext exactly once, in the 201 body.

Behavior
  • Plaintext is one-time. The reveal banner is the only place the full token appears in the UI. Refreshing the page, navigating away, or creating another token destroys it from client state.
  • Copy preserves provenance. Copy on a list card copies the masked preview (with prefix + lastFour); copy on the reveal banner copies the plaintext. The masked-preview-copy is intentionally useful for matching tokens to log lines.
  • Revoke confirms. Always via useConfirm. window.confirm is banned (docs/code-style.md).
  • Revoke is instant. The token is hashed-then-deleted on the server; any client using it starts receiving 401 within the next request.
  • Refresh after mutate. Both create and revoke call refresh() (re-fetches /api/agent-tokens) so the list reflects the new state without optimistic updates.
  • Admin only. Route is gated by middleware on the server; the avatar menu only shows the entry for admins. Members hitting the URL directly are 403'd.
  • Last-used timestamp. The server updates lastUsedAt on every authenticated API call by the token; the client renders it as relative time (relTime() helper in AgentsClient.tsx).
  • Reduced motion. No bespoke animations beyond color/border transitions, which collapse to 0 ms via the token override.
