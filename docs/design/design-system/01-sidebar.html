Draft sketch

Header + CTA placement — singular recipe

The Projects label lives inside the scrollable tree (no fixed pill above scroll). It scrolls together with project rows — same treatment as "No Project". A small + icon-button sits inline with the label for the canonical "New project" action. The upload affordance is the footer pill button — single canonical CTA, never duplicated in the header. The fixed-top pill variant and the header-menu CTA variant from earlier drafts are not part of the final design.

React API — composes with the sidebar shell

The full sidebar composes the logo + collapse button at the top, the scrollable tree in the middle (with the inline Projects label + project rows + "No Project" rows), and the footer with the canonical "New mockup" button. Tree expansion uses native CSS Grid transitions on grid-template-rows: 0fr → 1fr; no Radix Accordion needed for the tree.

// Sidebar.tsx — top-level shell, composes the tree + footer CTA
// uses native @radix-ui/react-collapsible IF we need controlled tree-expand state
<aside id="sidebar" data-collapsed={collapsed}>
  <Sidebar.Header>
    <Sidebar.Logo />
    <Sidebar.CollapseButton onClick={toggleCollapsed} />
  </Sidebar.Header>
  <Sidebar.Scroll>
    <Sidebar.SectionLabel>                       // inline label, scrolls with tree
      Projects
      <Sidebar.InlinePlusButton onClick={onNewProject}
        title="New project · Ctrl+Shift+P" />
    </Sidebar.SectionLabel>
    <Sidebar.ProjectTree projects={projects} />
    <Sidebar.SectionLabel>No Project</Sidebar.SectionLabel>
    <Sidebar.UnsortedMockups mockups={unsorted} />
  </Sidebar.Scroll>
  <Sidebar.Footer>
    <Sidebar.NewMockupButton onClick={onUpload}      // opens file picker → new-mockup-dialog
      title="Upload mockup · Ctrl+U">
      New mockup
    </Sidebar.NewMockupButton>
  </Sidebar.Footer>
</aside>
  • No Radix primitive backs the whole sidebar — it's an application shell. Individual pieces compose Radix: tree-expand can use @radix-ui/react-collapsible if needed; tooltips on the collapse button + + button use @radix-ui/react-tooltip; keyboard shortcut wiring is custom.
  • Inline Projects label is just a styled <div> inside the scroll container — no fixed-top sticky behaviour. Pros: simpler markup, matches "No Project" treatment, no flickering on scroll. Cons: when many projects scroll past, "Projects" label disappears off the top. Acceptable since the topbar breadcrumb already orients the user.
  • + icon-button is a custom slot, not a Radix DropdownMenu — there's no menu, just a single canonical "New project" action. Match the icon button visual from elsewhere in the project (kebab, plus, etc.) via a shared .btn-icon CSS class.
  • Footer button opens the system file picker via a hidden native <input type="file"> — same pattern as empty-state. On file selection, opens new-mockup-dialog pre-populated.
  • Tree-expand uses CSS Grid grid-template-rows: 0fr → 1fr with overflow:hidden on the inner — no JS height measurement, no Radix Accordion overhead.

Behavior

  • Click the collapse button: sidebar morphs to pill (84×40). Scroll + footer fade out before morph completes.
  • Click the pill: sidebar morphs back to 280px. Scroll + footer fade in after morph completes.
  • Click a project row: toggles its children (chevron rotates 90°, children expand via CSS Grid transition).
  • Click a mockup row: navigates to /projects/<slug>/.../<mockup>; row gains .active with --accent left bar.
  • Click + icon-button next to "Projects": opens a small "New project" dialog (single action, no menu).
  • Click the footer "New mockup" button: opens the system file picker. On selection, mounts new-mockup-dialog pre-populated.
  • Ctrl/Cmd+U opens the file picker globally. Ctrl/Cmd+Shift+P opens the New project dialog.
  • "Projects" label disappears off the top when the user scrolls deep into the tree — acceptable since the topbar breadcrumb already tells them where they are.
