Historic banner

Overlay pill anchored top-center of the viewer canvas (below the toolbar). Signals the viewer is rendering a past version of the mockup and offers a one-click return to current. Mounts when the user enters historic-version-viewing mode (URL ?v=<vid> drives the isHistoric flag on AppMainViewer). Carries no semantic status — it is a mode indicator, not an alert. For inline non-field notices use alert-banner; for transient confirmations use toast. No Radix primitive backs this component — custom at src/components/HistoricBanner/.

React API

No Radix primitive backs this component — Radix Primitives ships no inline-mode-indicator primitive (Toast is the closest cousin but is transient and floats top-right, see toast). Custom React component at src/components/HistoricBanner/HistoricBanner.tsx. The wrapper composes floatingSurface from src/styles/glass.module.css for the canonical blur+border treatment. 

// HistoricBanner.tsx — overlay pill, mode indicator
import { HistoricBanner } from '@/components/HistoricBanner/HistoricBanner';

// Mounted by AppMainViewer when ?v=<vid> is present
{isHistoric && (
  <HistoricBanner
    viewingLabel={`v${historicVersion}`}      // e.g. "v2"
    currentLabel={`v${currentVersion}`}        // e.g. "v5"
    onExit={() => router.replace(pathnameWithoutV)}
  />
)}

Notes:

  • viewingLabel + currentLabel — strings the caller formats. The component never derives version numbers itself; the parent owns the version model.
  • onExit — fires when the user clicks the trailing button. Caller typically strips ?v= from the URL via router.replace. The banner does not unmount itself — the parent's isHistoric guard re-evaluates on URL change and unmounts.
  • Outer wrapper is role="status" + aria-live="polite" (CUSTOM — no Radix equivalent). The mode change is non-blocking news, so polite SR announcement on mount is preferred over role="alert".
  • Exit button uses native :focus-visible for keyboard rings and data-tooltip for the tooltip text (consumed by the project's tooltip layer, see tooltip). No Radix Tooltip — that primitive is not installed.
  • No asChild slot pattern. The icon, label, and exit are fixed — the component is a single-purpose mode indicator, not a compound.

When to use which — historic-banner vs alert-banner vs toast
Use historic-banner when…Use the sibling when…
The user is in a persistent mode that diverges from the default view (here: viewing a past version) The user needs feedback about a transient action result — use toast (auto-dismiss, top-right, no anchor)
The signal should stay on screen the whole time the mode is active, anchored to the surface the mode affects The signal is a non-field message inside a form, dialog, or section — use alert-banner (inline, status hue, scoped to a surface)
The exit affordance is a one-click return to the default mode The message carries error / warning / success / info semantics — use alert-banner with the matching [data-status]

Behavior
  • Mounts when the parent's isHistoric guard is true (driven by URL ?v=<vid> on the mockup viewer). Unmounts when the guard flips back to false.
  • SR announces the mode switch on mount via role="status" + aria-live="polite" — a single polite phrase, not an alert.
  • Enter animation plays once on mount (opacity + 8px slide-down). Respects prefers-reduced-motion — duration collapses to 0ms with no JS guard.
  • Pointer click on the exit button fires onExit — caller typically strips ?v= from the URL. The banner does not self-unmount.
  • Keyboard Tab reaches the exit button (single focusable child). Enter / Space activate it. Native :focus-visible paints the accent ring; pointer clicks do not.
  • Tooltip on the exit button via data-tooltip="Back to current version" — consumed by the project's tooltip layer (see tooltip). Tooltip text mirrors the aria-label so SR and sighted users get the same affordance.
  • Anchored to the viewer canvas, not the viewport — scrolls with the canvas wrapper. Never floats over the topbar or sidebar.
  • Not dismissable on its own. The only way to remove it is to exit historic mode (via the exit button or by navigating away). This is deliberate — the banner is the source of truth that "you are seeing the past".
