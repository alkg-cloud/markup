02 — Breadcrumb

Navigational hierarchy strip rendered into the topbar at the top of every (app) route. Ships at src/components/Breadcrumbs/Breadcrumbs.tsx as a custom compound — no Radix primitive backs this surface (Radix focuses on overlays, not page chrome). Segments compose down from the current route via BreadcrumbSegment[]; the last segment is rendered as plain text with aria-current="page", every preceding segment is a next/link. When the chain exceeds three segments a middle ellipsis collapses the run; clicking it expands the full path inline without navigating.

React API — custom compound (no Radix primitive)

Radix Primitives deliberately does not ship a breadcrumb primitive — the surface is a layout strip, not an overlay or form, so it lives entirely as a custom compound at src/components/Breadcrumbs/Breadcrumbs.tsx. Consumers pass an array of BreadcrumbSegment objects; a segment without an href renders as the current page.

import { Breadcrumbs, type BreadcrumbSegment } from '@/components/Breadcrumbs/Breadcrumbs';

const segments: BreadcrumbSegment[] = [
  { label: 'lumen-coffee', href: '/projects/lumen-coffee' },
  { label: 'Landing',      href: '/projects/lumen-coffee/landing' },
  { label: 'Hero v3' },                                  // no href → current page
];

<Breadcrumbs segments={segments} />

Notes:

  • The component is a single export — no .Root / .Item / .Separator subcomponents. Segments are data, not children, because the chain is always computed from the route.
  • Truncation (custom). Radix has no breadcrumb truncation API. When segments.length > 3 the middle is collapsed to a single … button; clicking it sets internal showAll state and re-renders with every segment visible. No router navigation happens on the ellipsis.
  • Links use next/link. Caller doesn't wire a click handler — the component renders <Link href={…}> directly so Next.js prefetches the ancestor route on hover.
  • Current segment. The last segment, OR any segment with no href, renders as a <span> with aria-current="page". No focus, no hover — it's plain text.
  • Aria label. The wrapping <nav> uses aria-label="Navegação estrutural" (Portuguese — Markup's UI copy locale) which SR users announce as "Structural navigation, list, n items".

Behavior
  • Collapse threshold. When segments.length > 3, the middle n − 2 segments collapse into one … button. The first and last segments always render.
  • Expand on click. Clicking the ellipsis sets internal showAll state to true and the full chain renders in place — no route change, no Promise, no scroll.
  • Hover paints a pill. Link and ellipsis segments get a --surface-hover 2 px-padded background on hover so the affordance is clear without underlines.
  • Current truncates. The current segment is the only one with a hard max-width and ellipsis overflow. Ancestors stay short by convention (folder + project names are short); on mobile every ancestor link also truncates at 80 px so the chain stays one line.
  • Topbar layout. The component sits in Topbar with min-width:0; flex-shrink:1 so the centred finder-pill never gets pushed off-centre by a long crumb.
  • Empty segments array. Breadcrumbs returns null for segments.length === 0 (e.g. /all-projects root) — there's nothing meaningful to show.
  • Reduced motion. The colour/background transitions collapse to none under prefers-reduced-motion: reduce (per Breadcrumbs.module.css).
