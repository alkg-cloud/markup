03 — Finder Pill

Centred trigger in the topbar that opens the finder-overlay (a.k.a. command palette). Implemented inline inside src/components/Topbar/Topbar.tsx as .searchPill — a native <button>. No Radix primitive backs the pill; the overlay it opens is also custom (Radix Primitives ships no Command primitive — that's a cmdk-territory library and the project deliberately rolls its own to keep zero runtime deps for search). The label always reads Search…; the keyboard hint reflects the host platform via formatShortcut(['k']) → ⌘K on Apple, Ctrl+K elsewhere.

React API — custom button + dispatched CustomEvent

No Radix primitive — the pill is a native <button> inside Topbar. Click and ⌘K/Ctrl+K both dispatch a open-command-palette CustomEvent on document; the finder-overlay listens for it and opens. Decoupling via event keeps the Topbar from importing the palette and lets the keyboard shortcut work even when the palette is rendered inside an iframe (the mockup viewer rewires its contentDocument on every version switch).

// Topbar.tsx — pill + dispatch
import { VscSearch } from 'react-icons/vsc';
import { Kbd } from '@/components/Kbd/Kbd';
import { formatShortcut } from '@/lib/shortcuts/platform';

// Plain-text shortcut for aria-label: '⌘K' or 'Ctrl+K'
const kbdLabel = formatShortcut(['k']);

const handleSearchClick = () => {
  document.dispatchEvent(new CustomEvent('open-command-palette'));
};

<button
  type="button"
  className={styles.searchPill}
  aria-label={`Search... (${kbdLabel})`}
  onClick={handleSearchClick}
>
  <VscSearch size={16} aria-hidden="true" />
  <span className={styles.pillText}>Search...</span>
  <Kbd keys={['mod', 'k']} />   // renders [⌘][K] on mac, [Ctrl]+[K] on win/linux
</button>

Notes:

  • No primitive wrapper. Radix has no Button / SearchTrigger primitive. The pill is a stock <button type="button">; visual lives in Topbar.module.css under .searchPill.
  • Trigger via CustomEvent (custom). The CommandPalette mounts globally inside AppShell; both the pill click and the keyboard listener fire document.dispatchEvent(new CustomEvent('open-command-palette')) so the palette opens without prop drilling and without the Topbar importing it.
  • Platform-aware shortcut chip. <Kbd keys={['mod', 'k']} /> renders OS-aware keycap chips ([⌘][K] mac / [Ctrl]+[K] win/linux) from DS 29 — Kbd. The aria-label on the button still uses formatShortcut(['k']) for a plain-text SR announcement.
  • iframe rewiring. CommandPalette attaches the keyboard listener to every same-origin iframe document via a MutationObserver, so ⌘K works inside the mockup viewer even after its contentDocument reloads.
  • Centred via absolute. The pill is absolutely positioned (left:50%; transform:translateX(-50%)) inside the flex topbar so the centred placement survives a long breadcrumb on the left.

Behavior
  • Click → palette. Dispatches open-command-palette on document; the finder-overlay listens and mounts.
  • Keyboard shortcut. ⌘K on macOS, Ctrl+K elsewhere — registered by CommandPalette on document AND on every same-origin iframe document (rewired on iframe load via MutationObserver).
  • Platform-aware label. The kbd badge reflects the host OS via formatShortcut(['k']).
  • Centred under any breadcrumb. Absolutely positioned at left:50% so the centred placement survives long breadcrumb chains; nav and avatar flank it via flex.
  • Single overlay invariant. Opening the palette closes the avatar menu (the Topbar listens for the same event to dismiss its popover) so two overlays never stack.
  • No keyboard hint suppression. The kbd badge is always visible — the pill is wide enough at min-width 240 px to keep it; on narrower widths it would clip rather than hide (no current viewport hits this).
  • Reduced motion. Hover transitions collapse to 0ms via the token override.
