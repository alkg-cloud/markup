React API — compound component, Radix-aligned conventions

Radix Primitives ships @radix-ui/react-alert-dialog for modal alerts (already used by ConfirmDialog) but has no inline-banner primitive. This component is a custom compound that mirrors Radix's slot pattern so consumers feel at home: Root with status + Icon / Title / Description / Action / Close subcomponents, all accepting asChild via @radix-ui/react-slot.

// AlertBanner.tsx
import { Slot } from '@radix-ui/react-slot';

type Status = 'error' | 'warning' | 'success' | 'info';

<AlertBanner.Root status="error">
  <AlertBanner.Icon />                            // default per status, override via children
  <AlertBanner.Body>
    <AlertBanner.Title>Could not upload mockup</AlertBanner.Title>
    <AlertBanner.Description>Network request failed.</AlertBanner.Description>
  </AlertBanner.Body>
  <AlertBanner.Action asChild>                   // Slot — caller chooses element
    <button onClick={retry}>Retry</button>
  </AlertBanner.Action>
  <AlertBanner.Close asChild>                    // Slot — caller composes own X button
    <button aria-label="Dismiss"><XIcon /></button>
  </AlertBanner.Close>
</AlertBanner.Root>

Notes:

  • Root sets data-status on the wrapper; all CSS keys off this. Same recipe as Radix Toast (data-state, data-swipe).
  • aria-role is picked automatically: alert for error/warning, status for success/info. Override with role prop.
  • Action and Close use asChild + Slot so consumers control the element (button, link) and event handlers.
  • onOpenChange(open: boolean) on Root reports dismissal — caller manages mount/unmount (typical Radix controlled-pattern).
  • Auto-dismiss via duration={ms} prop on Root — mirrors Toast's API. Omit for sticky.
Buttons — Radix has no Button primitive

Worth calling out: @radix-ui/react-primitives deliberately does not ship a styled Button component — buttons are native <button> elements with project-owned styles. The existing ConfirmDialog follows this pattern: AlertDialog.Cancel asChild wraps a plain <button className={styles.cancel}>. We do the same here for Action and Close. So:

  • Banner's Action / Close = Slot wrapping a native <button>.
  • Visual button variants (.action pill here; .btn-secondary, .btn-accent, .btn-danger elsewhere) live in CSS, not in TS components.
  • Where the consumer needs Radix semantics (focus trap, escape, portal), they wrap our button with the right Radix Root — same recipe as ConfirmDialog.

Behavior
  • error + warning dismiss only on explicit close or programmatic unmount — never auto-dismiss.
  • success + info may auto-dismiss; pass duration to Root.
  • SR announces error / warning on insertion (role="alert"). success / info use polite role="status".
  • Multiple banners stack vertically (gap from parent).
  • Action button receives focus on mount when present (typical recovery affordance).
  • Pairs with input-field error state — both can coexist (global banner + per-field flags).
