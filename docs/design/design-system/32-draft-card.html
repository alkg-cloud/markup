32 — DraftCard

Inline annotation composer that lives at the top of the annotations-rail while a user is drafting. Replaces the previous modal annotation-composer surface. Owns the textarea, the action row (Cancel · Draft ⌘S · Send ⌘↵), the live pin-count microcopy, and the status footer (unsaved · saving · saved · sending · error). Persists per (mockupId, userId) in localStorage — never on the server. Companion: see pins for the draft pin (data-status="pending") visual that lives on the canvas for the duration of the draft.

Eight states share one CSS recipe. Variants are driven by three orthogonal attributes on the wrapper: data-status (unsaved / saving / saved / sending / error) drives the status footer microcopy + dot hue; data-body-state (empty / typing / over-limit) gates the Send button + char counter visibility; data-pin-count (zero / some / max) drives the pin-row copy. No modifier classes — .draft-card-error and friends do not exist.

Discard draft?
You'll lose 3 pins and the text you typed. This can't be undone.
Cancel discards silently when the draft is empty. With body OR pins present, the action opens a Radix AlertDialog confirming the destructive intent. The body line is templated: "You'll lose N pins and the text you typed." Returning via Keep editing restores the draft active; Discard clears localStorage and animates the DraftCard out.

The DraftCard mounts in its own sticky slot at the top of the rail's expanded body, between .head and the scrollable .list of published AnnotationCards. The list scrolls independently — the DraftCard's action row never leaves the viewport. When a draft is active the rail force-expands to 300 px without the pinned accent ring; the user's pre-existing pin/unpin preference is preserved underneath and restored on send or discard.
React API — composes Radix Form + AlertDialog

Custom in-project component (src/components/DraftCard/DraftCard.tsx) — no single Radix primitive covers the surface. The textarea is wrapped in @radix-ui/react-form so consumers get standard form events and constraint-validation hooks. The Cancel action opens a @radix-ui/react-alert-dialog. The state machine (null ↔ active ↔ sent / discarded) lives in the parent AppMainViewer via useDraftPersistence(mockupId, userId) — a custom hook that owns localStorage, the 800 ms debounce, and stale-draft cleanup. 

// DraftCard.tsx — composes Radix Form for the textarea
import * as Form from '@radix-ui/react-form';
import * as AlertDialog from '@radix-ui/react-alert-dialog';
import { Kbd } from '@/components/Kbd';
import styles from './DraftCard.module.css';

// status drives the footer microcopy + dot hue
type DraftStatus = 'unsaved' | 'saving' | 'saved' | 'sending' | 'error';
type BodyState = 'empty' | 'typing' | 'over-limit';
type PinCount = 'zero' | 'some' | 'max';

<Form.Root onSubmit={handleSend} className={styles.draftCard}
  data-status={status}
  data-body-state={bodyState}
  data-pin-count={pinCount}
  data-density={'compact'}>                              // 'compact' is the DS default
  <div className={styles.draftHeader}>
    <span className={styles.draftMarker}>Draft</span>
    <span className={styles.draftSpacer} />
    {bodyState !== 'empty' && (
      <span className={styles.draftCountChars}>{body.length.toLocaleString()} / 10 000</span>
    )}
  </div>

  <Form.Field name="body">
    <Form.Label className="sr-only">Annotation body</Form.Label>
    <Form.Control asChild>
      <textarea
        className={styles.draftTextarea}
        value={body}
        onChange={(e) => setBody(e.target.value)}
        placeholder="What's wrong here? What would you change?"
        required maxLength={10_000}
        autoFocus
      />
    </Form.Control>
  </Form.Field>

  <div className={styles.draftPinRow}>
    <span className={styles.pinMini} aria-hidden />
    <strong>{pins.length} {pins.length === 1 ? 'pin' : 'pins'}</strong>
    <span className={styles.hint}>{pinHintCopy(pins.length, pinCount)}</span>
  </div>

  <div className={styles.draftActions}>
    <CancelButton hasContent={body.length > 0 || pins.length > 0} onConfirm={onDiscard} />
    <button type="button" className={cn(styles.btn, styles.btnSecondary)}
            disabled={!hasUnsavedChanges} onClick={onSaveDraft}>
      Draft <Kbd keys={['mod', 's']} />
    </button>
    <Form.Submit asChild>
      <button className={cn(styles.btn, styles.btnPrimary)}
              disabled={bodyState !== 'typing'}>
        {status === 'sending' ? 'Sending…' : status === 'error' ? 'Retry' : 'Send'}
        <Kbd keys={['mod', 'enter']} />
      </button>
    </Form.Submit>
  </div>

  <div className={styles.draftStatus} role="status" aria-live="polite">
    <span className={styles.dot} />
    {statusCopy(status, lastSavedAt)}
  </div>
</Form.Root>

// CancelButton wraps the AlertDialog so the parent stays declarative
function CancelButton({ hasContent, onConfirm }) {
  if (!hasContent) return <button className={styles.btnGhost} onClick={onConfirm}>Cancel</button>;
  return (
    <AlertDialog.Root>
      <AlertDialog.Trigger asChild>
        <button className={styles.btnGhost}>Cancel</button>
      </AlertDialog.Trigger>
      <AlertDialog.Portal>
        <AlertDialog.Overlay className={styles.alertOverlay} />
        <AlertDialog.Content className={styles.alert}>
          <AlertDialog.Title>Discard draft?</AlertDialog.Title>
          <AlertDialog.Description>
            You'll lose <strong>{pinCountLabel}</strong> and the text you typed.
          </AlertDialog.Description>
          <div className={styles.alertActions}>
            <AlertDialog.Cancel asChild><button>Keep editing</button></AlertDialog.Cancel>
            <AlertDialog.Action asChild><button onClick={onConfirm}>Discard</button></AlertDialog.Action>
          </div>
        </AlertDialog.Content>
      </AlertDialog.Portal>
    </AlertDialog.Root>
  );
}

Notes:

  • Radix Form backs the textarea. Form.Root + Form.Field + Form.Control asChild give us the constraint-validation pipeline (Form.Message can render server-side errors next to the field). Form.Submit asChild is the Send button.
  • AlertDialog wraps Cancel when the draft is non-empty. When empty, Cancel is a bare button — no dialog needed.
  • Kbd is the DS component from 29-kbd.html. It auto-translates the mod key into ⌘ / Ctrl per useIsMac().
  • (custom) draft persistence — there is no Radix or library primitive for "save my form to localStorage". useDraftPersistence is project-owned. It reads on mount, debounces 800 ms on changes, flushes on visibilitychange / beforeunload, and discards stale drafts > 30 days old.
  • (custom) status state machine — Radix has no draft-status concept. The five status values (unsaved / saving / saved / sending / error) and their hue mapping live in the project.
  • (custom) pin lifecycle. The DraftCard does not own the canvas-side pin rendering — that's the PinLayer. The parent AppMainViewer threads onPinAdd / onPinRemove between the two; the DraftCard only displays the count.
  • One draft per (mockupId, userId). Multi-draft is not supported. Restoring a stored draft on mount transitions null → active automatically.

unsaved Our [data-status="unsaved"] — set immediately on any body or pin change; cleared when the debounce flush completes. (custom) Status hue --warning. Microcopy: Unsaved changes or Type to start (empty). role="status" on the footer.
saving Our [data-status="saving"] — during the localStorage flush (debounce window or explicit ⌘S). (custom) Status hue --text-dim. Pulsing accent dot. Microcopy: Saving… aria-live="polite".
saved Our [data-status="saved"] — when the most recent change has been flushed to localStorage. Save button disables. (custom) Status hue --success. Microcopy: Draft saved Xs ago (relative time, ticks every 30 s). role="status".
sending Our [data-status="sending"] — between Send click / ⌘↵ and POST response. Card opacity 0.85; all inputs locked. (custom) Status hue --accent-bright. Pulsing accent dot. Microcopy: Sending…. Send button label becomes Sending…. aria-busy="true" on the card.
error Our [data-status="error"] — after POST 4xx/5xx (validation, network, server). Draft stays in localStorage. (custom) Status hue --danger. Microcopy: Couldn't send · saved locally (or the field-specific error). Send button label becomes Retry. aria-live="assertive" on the footer.
empty (body) Our [data-body-state="empty"] — body length is 0. (custom) Char counter hidden. Send button disabled (opacity 0.4; pointer-events none). aria-disabled="true" on Send.
over-limit (body) Our [data-body-state="over-limit"] — body length > 10 000. Pairs with Radix Form.Message match="tooLong" for the error text. (custom) Char counter hue --danger. Textarea border --danger-border. Send disabled. aria-invalid="true" on the textarea.
zero (pins) Our [data-pin-count="zero"] — no pins. (custom) Pin row: 0 pins · click on the mockup to add. Send remains enabled (pins are optional). —
max (pins) Our [data-pin-count="max"] — 20 pins reached. (custom) Pin row hint hue --warning: max reached. The 21st canvas click is consumed and toasts Maximum 20 pins. —
The three attributes (data-status, data-body-state, data-pin-count) are orthogonal — every combination is rendered correctly. Each row's "(custom)" flag means there is no Radix primitive that owns the state; the project does. Send-disabled is driven by data-body-state; Save-disabled by data-status="saved"; the 21-pin guard lives in the canvas click handler, not on the DraftCard itself.
When to use which
Use DraftCard when…Use AnnotationCard when…
The user is composing a new top-level annotation that hasn't been sent yet. The annotation is persisted (status open / needs review / resolved) and the user is reviewing or replying.
The card needs to own a textarea, action buttons, and a status footer. The card is read-only summary (author, body, status pill, reply count) with thread-deep navigation on click.
Lifecycle is active → sent | discarded and lives in localStorage. Lifecycle is server-owned; status changes via PATCH /api/annotations/[id].
DraftCard and AnnotationCard cohabit the rail's expanded body. DraftCard pins to the top sticky slot; AnnotationCard fills the scrollable .list. They share the rail's glass-surface tokens but otherwise have separate recipes. The deprecated annotation-composer modal is removed in this change-set — consumers that imported AnnotationComposer migrate to DraftCard mounted by AnnotationsRail.

Behavior
  • Entry. Mounts when the parent AppMainViewer transitions draft state from null → active. Triggers: + New annotation button on the rail, single-key N when no input is focused, or auto-restore on viewer mount when localStorage[markup:draft:<mockupId>:<userId>] holds a non-stale draft.
  • Mount-time focus. The textarea receives focus on mount. Restored drafts focus the textarea but place the caret at the end.
  • Force-expand the rail. While the DraftCard is mounted, the parent rail forces its expanded state without showing the pinned accent ring. The user's pre-existing lock-open state is preserved underneath and restored on send / discard.
  • Auto-save. 800 ms debounce on body or pins change. Flush on visibilitychange (tab hidden) and beforeunload (best-effort). Stores { body, pins, lastSavedAt, schemaVersion: 1 }.
  • Explicit save. ⌘+S (Mac) / Ctrl+S (Win) flushes immediately and updates lastSavedAt. preventDefault suppresses the browser's save-page action.
  • Send. ⌘+↵ or button. Disabled when data-body-state is empty or over-limit. POST /api/mockups/:id/annotations with { body, anchors: pins, colorIndex }. Optimistic insert into the published list. On success: clear localStorage, unmount the DraftCard, restore rail's prior pin state.
  • Cancel. Esc or button. If body and pins are both empty: discard silently. Otherwise open the cancel-confirm AlertDialog. Confirm → clear localStorage + animate out (200 ms). Cancel-of-cancel → return to active.
  • Canvas pin sync. The DraftCard renders only the count; canvas-side pin add/remove is owned by AppMainViewer + PinLayer. Adding a pin past 20 is consumed at the dispatcher (no pin appended) and surfaces a toast — the DraftCard reflects the data-pin-count="max" state.
  • Stale-draft cleanup. On viewer mount, if lastSavedAt is > 30 days old, the entry is discarded without prompting.
  • Single-draft invariant. One draft per (mockupId, userId). The parent's draft state holds at most one entry; the entry triggers N while a draft is active focuses the existing DraftCard's textarea instead of creating a new one.
  • Reduced motion. The DRAFT marker pulse, status-dot pulse, and save / sending animations all collapse to 0 ms under prefers-reduced-motion: reduce via the global token override.
