18 — Annotations Rail

Left-side floating panel inside the mockup viewer (src/components/AnnotationsRail/AnnotationsRail.tsx). Custom compound — no Radix primitive. Three states: collapsed (60 px badge column), hover-expanded (300 px transient on mouseenter), pinned-expanded (300 px sticky via lock-open). Drag handle (top) uses GoGrabber rotated 90°; add button morphs from 32 px circle to full-width pill on expand. Glass surface composes the same recipe as dialog, popover, and toolbar — 16 px blur, --border, 14 px radius, --shadow-popover. Cards in the expanded list are AnnotationCard instances; the add button fires onCreate on the parent canvas. Capped at 70 vh / 80 vh on mobile.

React API — custom compound (no Radix primitive)

Backed by the custom <AnnotationsRail> component (src/components/AnnotationsRail/AnnotationsRail.tsx). No matching Radix primitive — the floating, draggable, hover-vs-pin behaviour is too app-specific. Internal state covers hover-expanded (transient on mouseenter), pinned-on (sticky via the lock toggle), and dragging (while the grab handle is held). The parent canvas owns the badge data and renders the expanded card list as children — the rail is a layout shell, not a data store.

// Render at the AppMain level — boundsRef = the mockup canvas container
import { AnnotationsRail } from '@/components/AnnotationsRail';
import { AnnotationCard } from '@/components/AnnotationCard';

function MockupViewer({ annotations, activeId, setActiveId, canvasRef }) {
  const badges = annotations.map((a, i) => ({
    annotationId: a.id,
    colorIndex: a.colorIndex,
    label: i + 1,
  }));

  return (
    <AnnotationsRail
      boundsRef={canvasRef}                            // draggable position clamped to this rect
      badges={badges}                                  // summary list for collapsed-state column
      activeAnnotationId={activeId}                    // adds .active on the matching badge + card
      onBadgeClick={setActiveId}                       // click a badge → activate annotation
      onCreate={() => createAnnotation()}              // + button → consumer-owned create flow
      count={annotations.length}                       // header pill, default = badges.length
      resetPositionKey={isFullscreen ? 'fs' : 'win'}    // reset drag offset when bounds change
      expandSignal={pinFromOutside}                    // bumped from parent to pin programmatically
    >
      {annotations.map((a) => (
        <AnnotationCard
          key={a.id}
          annotation={a}
          active={a.id === activeId}
          onClick={() => setActiveId(a.id)}
        />
      ))}
    </AnnotationsRail>
  );
}

// Props the rail owns internally — never expose these:
//   - hover-expanded state (mouseenter / mouseleave on the body)
//   - pinned state (toggled by the lock button)
//   - drag state (mousedown on .grab, mousemove on document)
//   - draggedPosition (top/left after a drag, clamped to boundsRef)
// The parent only controls semantic data (badges, active id, onCreate).

Notes:

  • No Radix primitive. The hover-vs-pin two-mode expand pattern, the integrated drag handle (clamped to a boundsRef), and the morphing add button are too domain-specific for a generic library. Each piece (drag, hover-expand, lock-open) lives inline in the component.
  • Drag handle uses GoGrabber from react-icons/go at 25 × 25 rotated 90° so the dots run horizontally — same affordance as the toolbar (19-toolbar.html).
  • expandSignal is a numeric token that the parent bumps to pin the rail open from outside (e.g. when a new annotation is created, the canvas wants the user to see it in context). The rail watches the value, not its content — passing the same number twice is a no-op.
  • resetPositionKey clears the drag offset when the containing block changes (fullscreen toggle changes the bounds). Without this, a previously-dragged rail can end up off-screen.
  • The expanded card list is a slot — the parent renders <AnnotationCard> children. The rail doesn't know about the card data shape, so cards can evolve independently.
  • Add button is native <button> with the .add class — no Radix wrapping (Radix has no Button primitive). The collapsed → expanded morph is pure CSS (width + .add-label display).
  • Shortcut chip via <Kbd>. The add button label renders <Kbd keys={['mod','shift','n']} /> from DS 29 — Kbd. The old .shortcut span is removed; aria-label on the button uses formatShortcut(['shift','n']) for a plain-text SR announcement.
State decision matrix
StateTriggerVisualAria
collapsed Default. No hover, lock unpinned. 60 px column. Badge stack visible. Add button is a 32 px circle. —
hover-expanded mouseenter on the rail body. Width animates to 300 px (--motion-base / --ease-spring). Cards list replaces the badge column. Add button morphs to full-width pill with label + shortcut. —
pinned-expanded Lock-open button clicked, or expandSignal bumped from parent. Same as hover-expanded but border becomes --accent-overlay-mid with an extra 1 px accent overlay ring. Persists after mouseleave. Lock button: aria-pressed="true".
dragging mousedown on the grab handle. Document captures mousemove; rail position updates within boundsRef. Cursor changes to grabbing. —
empty Zero annotations in the expanded view. Cards list slot becomes a muted "No annotations yet" message + shortcut hint. Add button still present. —
badge-active activeAnnotationId matches a badge's id. Badge gets a 4 px coloured glow ring + brighter fill; matching card gets .active (accent-overlay bg + 2 px left bar). Matching card: aria-current="true".
has-draft Orthogonal to expanded/pinned. Set when a DraftCard is mounted via the draft prop. Rail force-expands to 300 px without the pinned accent ring (the user's pre-existing pin state is preserved underneath). A .draft-slot renders the DraftCard in a sticky position at the top of the expanded body, above the scrollable .list. In the collapsed view, a sentinel .badge.draft (dashed accent +, pulsing) appears at the top of the badge column. The DraftCard owns its own aria-live announcements.
"hover-expanded", "pinned-expanded", and "dragging" are internal to the rail — the parent only sets expandSignal for programmatic pin. The drag, hover-expand, and lock toggle are all owned by the component.

Behavior
  • Hover expand. mouseenter on the rail body triggers expand; mouseleave collapses back — unless pinned. Width animates with --motion-base / --ease-spring.
  • Lock-open (pin). The lock button toggles a sticky pin. While pinned, mouseleave doesn't collapse the rail. Border + ring shift to accent.
  • Drag-to-reposition. mousedown on the grab handle starts a drag. The rail position updates within boundsRef (clamped to keep at least 8 px on every edge).
  • Reset on bounds change. resetPositionKey change clears the drag offset (rail returns to spec defaults). The canvas bumps this when fullscreen toggles.
  • Programmatic pin. expandSignal bumps from parent pin the rail open (used after creating a new annotation so the user sees it land in the list).
  • Badge-card sync. Setting activeAnnotationId highlights both the badge (in the collapsed column) and the card (in the expanded list).
  • Add button. Native <button>. Morphs from 32 px circle (collapsed) to full-width pill (expanded) — same transition tokens as the rail width.
  • Keyboard. Single-key N (rendered on the add button) fires onCreate from anywhere in the canvas — registered globally by the parent and suppressed while any input/textarea/contenteditable is focused so the letter still types normally.
  • DraftCard slot. When the parent passes a draft prop, the rail mounts a DraftCard in a sticky .draft-slot at the top of the expanded body. While the slot is filled, the rail force-expands (no accent ring) and the footer add-button hides — the DraftCard is the entry. On send/discard, the rail returns to the user's prior pinned/unpinned preference.
  • Reduced motion. Under prefers-reduced-motion: reduce, both the width transition and the add-button morph collapse to a 0 ms snap via the token override at the top of the file.
