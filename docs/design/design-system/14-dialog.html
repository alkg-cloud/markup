14 — Dialog

Modal dialog primitive backing every form, info, or confirmation overlay in the app. Implemented as src/components/Dialog/RadixDialog.tsx — a thin compound wrapper over @radix-ui/react-dialog for non-destructive overlays, with useConfirm in src/components/ConfirmDialog/ConfirmDialog.tsx wrapping @radix-ui/react-alert-dialog for destructive confirms. Shares the glass-surface standard with rail, toolbar, popovers and tooltips: --surface-glass-bg, blur 16 px / saturate 140%, glass border, 14 px radius, --shadow-popover. Form bodies compose input-field for per-field validation and alert-banner for non-field messages.

React API — composes Radix Dialog + Radix AlertDialog (for confirms)

Two Radix primitives back this DS entry. @radix-ui/react-dialog powers the regular form/info dialog via the thin RadixDialog compound (open/close controlled by the consumer). @radix-ui/react-alert-dialog powers the useConfirm hook for destructive confirms — it enforces an explicit choice and uses role="alertdialog" for SR. Form bodies wrap content in @radix-ui/react-form + InputField for ValidityState-driven validation.

// 1) Regular form/info dialog — RadixDialog wraps @radix-ui/react-dialog
import { RadixDialog } from '@/components/Dialog/RadixDialog';
import { InputField } from '@/components/InputField';
import * as Form from '@radix-ui/react-form';

<RadixDialog.Root open={open} onOpenChange={setOpen}>
  <RadixDialog.Portal>
    <RadixDialog.Overlay />                       // scrim — blur(16px) + --scrim-glass-bg
    <RadixDialog.Content>                       // glass surface — 14px radius, --shadow-popover
      <RadixDialog.Title>New project</RadixDialog.Title>

      <Form.Root onSubmit={handleSubmit}>            // composes @radix-ui/react-form
        <InputField.Root name="name">             // see 28-input-field.html
          <InputField.Label>Project name</InputField.Label>
          <InputField.Control asChild>
            <input type="text" required pattern="[A-Za-z0-9_-]+" />
          </InputField.Control>
          <InputField.Message match="patternMismatch">
            Use letters, digits, hyphens, or underscores only.
          </InputField.Message>
        </InputField.Root>

        <div className={styles.actions}>
          <RadixDialog.Close asChild>                // Slot — caller owns the button
            <button type="button" className={styles.btnSecondary}>Cancel</button>
          </RadixDialog.Close>
          <Form.Submit asChild>
            <button type="submit" className={styles.btnAccent}>Create</button>
          </Form.Submit>
        </div>
      </Form.Root>
    </RadixDialog.Content>
  </RadixDialog.Portal>
</RadixDialog.Root>

// 2) Destructive confirm — useConfirm wraps @radix-ui/react-alert-dialog
import { useConfirm } from '@/components/ConfirmDialog/ConfirmDialog';

function ProjectActions() {
  const { confirm, dialog } = useConfirm();           // `dialog` mounts once in tree

  async function handleDelete() {
    const ok = await confirm({                       // Promise<boolean>
      title: 'Delete project "lumen-coffee"?',
      description: 'This cannot be undone.',
      confirmLabel: 'Delete',
      danger: true,                                    // → .btn-danger styling
    });
    if (ok) await deleteProject();
  }

  return <> { dialog } <button onClick={handleDelete}>Delete</button> </>;
}

Notes:

  • RadixDialog re-exports Root / Trigger / Portal / Close stock and wraps Overlay / Content / Title / Description with the dialog CSS module — see src/components/Dialog/RadixDialog.tsx.
  • The dialog auto-focuses the first focusable element inside Content on open — no autoFocus needed on inputs (and Biome's a11y rule forbids it).
  • RadixDialog = regular dialog: Escape closes, click-outside closes, controlled by open + onOpenChange. Use for forms (create / edit / settings) and pure-info dismiss-only dialogs.
  • useConfirm = destructive confirm: Promise-based — resolves true on Confirm, false on Cancel / Escape / scrim-click. Mount dialog once in the consumer's tree so the portal has a root. See src/components/ConfirmDialog/ConfirmDialog.tsx.
  • Never use window.confirm / alert / prompt — they ignore tokens and hijack touchscreens. useConfirm is the canonical replacement.
  • Custom button classes (.btn-secondary, .btn-accent, .btn-danger) — Radix ships no Button primitive. Wrap native <button> in asChild slots (RadixDialog.Close, Form.Submit, AlertDialog.Cancel, AlertDialog.Action).
State decision matrix
StateTriggerVisualAria
closed open={false} — not mounted (Radix unmounts via Portal). Nothing in the DOM. Scrim absent. —
open open={true} on RadixDialog.Root. Scrim fades in; dialog scales 0.96 → 1 + translateY 8 → 0 over --motion-base / --ease-spring. First focusable autofocused. role="dialog", aria-modal="true", aria-labelledby = Title id.
confirming (custom) useConfirm().confirm({ … }) — Promise pending. Same surface; danger: true swaps primary button to .btn-danger. Cancel and Confirm are required choices (no implicit dismiss-to-true). role="alertdialog", aria-describedby = Description id when present.
submitting (custom) Consumer flips loading after submit; we disable Cancel + primary. Both action buttons :disabled (opacity 0.4, cursor:not-allowed). Dialog remains open until the request resolves. Disabled buttons are SR-announced as disabled via native semantics.
"confirming" and "submitting" are custom states — Radix Dialog / AlertDialog don't model them. They live as consumer-managed props (danger, loading) and surface visually through the button variant classes.
When to use which
PrimitiveUse forRadix backing
RadixDialog Forms (create / edit project, folder, mockup), settings panels, info / preview overlays. Caller controls open; Escape and scrim-click dismiss. @radix-ui/react-dialog
useConfirm / ConfirmDialog Destructive confirms (delete project, delete mockup, discard unsaved changes). Promise-based — resolves true only on explicit Confirm. @radix-ui/react-alert-dialog
alert-banner Inline non-modal notices inside a dialog body or page section (rate limit, server error, success). Pair with either primitive above. Custom — see 27-alert-banner.html
input-field Per-field sync validation inside a dialog form body. Composes @radix-ui/react-form. See 28-input-field.html
Rule of thumb: if the user can dismiss the overlay with no consequences, use RadixDialog. If dismissing the overlay must mean no (the destructive action does not run), use useConfirm.

Behavior
  • Focus trap. Radix Focus Scope locks Tab inside Content while open and returns focus to the trigger on close.
  • Autofocus. First focusable element inside Content is autofocused on open. No autoFocus attribute needed (Biome's a11y rule forbids it).
  • Escape closes. RadixDialog dismisses on Escape; useConfirm resolves false on Escape (treated as Cancel).
  • Click-outside closes. Scrim click triggers onOpenChange(false) on RadixDialog; same Cancel behaviour for useConfirm.
  • Scroll lock. Body scroll is frozen while the overlay is mounted (Radix Remove-Scroll).
  • Portal mount. Content renders at document.body via RadixDialog.Portal / AlertDialog.Portal — never trapped in a transformed ancestor.
  • Aria wiring. Title + Description ids are auto-bound to aria-labelledby / aria-describedby. Pass aria-describedby={undefined} to Content when the dialog has no Description child (Radix warns otherwise).
  • Submitting. Consumer disables Cancel + primary while a request is in flight; dialog stays mounted until the Promise resolves (success closes; failure surfaces an alert-banner inside the body).
  • Reduced motion. Under prefers-reduced-motion: reduce, the enter animation collapses to a 0 ms opacity flip (token override at top of file).
