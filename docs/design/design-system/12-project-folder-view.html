12 — Project / Folder View

The browse surface for project roots and any folder within. Implemented as src/app/(app)/projects/[slug]/page.tsx (project root) and src/app/(app)/projects/[slug]/[...path]/page.tsx (nested folders) — both share the same ProjectContent client component at src/app/projects/[slug]/ProjectContent.tsx. Header (FolderHeader) shows project/folder name + item count; body is a responsive grid of folders + mockups; empty-state replaces the grid when there are no items. Cards compose project-card-style hover and link semantics.

React API — composition page (no Radix primitive)

Page-level composition. Both routes (page.tsx root + [...path]/page.tsx nested) are client components that fetch a payload from GET /api/projects/by-slug/[slug]/view and forward it to the shared ProjectContent component. No Radix primitive backs the layout itself — it composes AppMain, Topbar (which embeds the avatar-menu), FolderHeader, FolderCard, and MockupCard.

// src/app/projects/[slug]/ProjectContent.tsx — abridged
import { AppMain } from '@/components/AppMain/AppMain';
import { Topbar } from '@/components/Topbar/Topbar';
import { FolderHeader } from '@/components/FolderHeader/FolderHeader';
import { FolderCard } from '@/components/FolderCard/FolderCard';
import MockupCard from '@/app/mockups/MockupCard';
import { UploadEmptyState } from '@/components/EmptyState/UploadEmptyState';
import { useNewMockupDialog } from '@/components/NewMockupDialog';

const isEmpty = folders.length === 0 && mockups.length === 0;
const { openDialog } = useNewMockupDialog();

return (
  <div style={{ display: 'flex', flexDirection: 'column', flex: 1 }}>
    <Topbar breadcrumbs={breadcrumbs} userName={…} userRole={…} />
    <AppMain variant="scroll" ariaLabel="Project workspace">
      <FolderHeader
        icon={folderName ? null : projectIcon}                  // emoji or null
        name={folderName ?? projectName}
        count={folders.length + mockups.length}
      />
      {isEmpty ? (
        <UploadEmptyState context={folderName ? 'folder' : 'project'} … />
      ) : (
        <div style={gridStyle}>
          {folders.map((f) => <FolderCard key={f.id} folder={f} projectSlug={…} folderPath={…} />)}
          {mockups.map((m) => <MockupCard key={m.id} {...m} href={mockupSlugHref(…)} />)}
        </div>
      )}
    </AppMain>
  </div>
);

Notes:

  • One ProjectContent, two routes. The project-root and nested-folder pages differ only in the payload they fetch (/view at root vs /view?path=… nested). The view layer is unified — folder-name vs project-name is just a prop swap.
  • Client-side rendering. Per CLAUDE.md the entire page tree is 'use client'. Data is fetched in useEffect; 401 → window.location.replace('/login'), 404 → notFound(), error → <ErrorState />, loading → <LoadingState />.
  • Card hrefs. Folders use folderHref(projectSlug, folderPath); mockups use mockupSlugHref(projectSlug, folderPathNames, slug) — both from @/lib/project/routes. Path-based, not ID-based, so URLs are human-readable and survive renames.
  • New Mockup button. Wired to useNewMockupDialog().openDialog({ file?, target }) — opens new-mockup-dialog with the resolved {projectId, folderId} pre-filled.
  • New Folder button. Wired to useNewFolderDialog() (similar pattern). At the empty-state, file-drop bypasses the dialog and opens it directly with the file pre-attached.
  • Topbar embedding. The page is responsible for mounting Topbar, including passing userRole so the avatar menu can conditionally show admin items.
State decision matrix
StateTriggerVisualAria
loading Mount; fetch in flight. <LoadingState /> (spinner). Loading region announced via role/aria on LoadingState.
populated Fetch returned ≥1 folder or mockup. Header + grid of FolderCard + MockupCard. AppMain has ariaLabel="Project workspace".
empty Fetch returned 0 folders + 0 mockups. UploadEmptyState in place of the grid; accepts file-drop. Drop-zone has aria-label.
not_found 404 from API (project slug doesn't exist or no access). notFound() → Next.js renders not-found.tsx. —
error Non-2xx, non-404, non-401 response. <ErrorState error="Failed to load project." />. role="alert" on the error component.
unauthorized 401 response. Immediate redirect to /login via window.location.replace. —
The page itself owns all five states. ProjectContent never sees loading, not_found, error, or unauthorized — the route file gatekeeps and only mounts ProjectContent on the ok branch.

Behavior
  • Ordering. Folders are rendered before mockups in the grid; within each group the API returns its native order (folders alphabetical, mockups by updatedAt desc).
  • URL stability. Folder URLs are path-based (/projects/lumen-coffee/landing/hero) not ID-based, so renames break links — by design, since folder slugs are url-safe-name validated.
  • Drop target. The empty state accepts file-drop (a .zip) and hands the file to useNewMockupDialog().openDialog({ file, target }) with {projectId, folderId} pre-filled. Non-empty views currently do not accept drops — uploads go through "New Mockup".
  • Per-card actions. Folder and mockup cards expose a kebab menu on hover with rename/move/delete; see sidebar for the kebab pattern. Deleting a folder cascades to its descendants and prompts via useConfirm.
  • Breadcrumbs. The topbar's breadcrumb reflects the full path from project root to current folder; the in-view header only shows the current folder name. No duplication.
  • Auth. Pages are wrapped by middleware cookie-check + a client-side useIdentity() call inside the page. 401 → redirect to /login; the page never tries to render unauthenticated content.
  • Reduced motion. No bespoke animations beyond color/border transitions on cards, which collapse to 0 ms via the token override.
