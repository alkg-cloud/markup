06 — Icon Picker

Four-tab grid picker used inside the new-project-dialog to choose a project glyph. Ships at src/components/IconPicker/IconPicker.tsx as a custom compound — no Radix primitive backs the picker (Radix has no Tabs+Grid composite; the project deliberately avoids Radix Tabs here because the tab change must also reset the search query). Emits stable string tokens (vsc:VscFile, brand:react, ui:layers, emoji:🎨) that project-card resolves back to the rendered glyph via resolveIconToken(). Tabs are code / brands / ui / emoji; the active tab carries an underline at --accent-bright. Multi-set palette: Code from react-icons/vsc (Codicons), Brands from react-icons/si (Simple Icons), UI from react-icons/lu (Lucide), Emoji as native glyphs. Icon cells render real React components — no dangerouslySetInnerHTML.

A — Default (Code tab, no selection)
Tabs sit above the search row in the live component (the legacy DS had them below — this snapshot restores the shipping order). Footer hides when nothing is selected (value === undefined); the popover collapses to a slimmer max-height.
B — Selection + token footer
vsc:VscFile
Selected cell is outlined in --accent with an --accent-overlay-soft fill and the glyph painted at --accent. Footer reveals once value is set and shows the canonical token string the consumer receives.
C — Filtered (search active)
vsc:VscFolder
Search filters within the active tab only. Switching tabs resets the search query to empty — the picker rebuilds its grid from the new tab's index.
D — Emoji tab
emoji:🔥
Emoji tab swaps SVG icons for native glyphs at 14 px / line-height 1. Same selection visuals — the border + tint apply regardless of cell content.
React API — custom compound (no Radix primitive)

No Radix primitive. The picker is a single export taking value + onSelect — the controlled string token. filterIcons(tab, search) (in ./icons.ts) returns the in-tab + matching set; cells render either a react-icons component (for Code / Brands / UI sets) or a native emoji glyph (for the Emoji set) — no dangerouslySetInnerHTML. The grid uses ARIA listbox + option semantics so screen readers describe it as a selectable list.

import { IconPicker } from '@/components/IconPicker/IconPicker';

<IconPicker
  value={icon}                         // 'vsc:VscFile' | 'emoji:🔥' | undefined
  onSelect={(token) => setIcon(token)}
/>

Notes:

  • Why not Radix Tabs (custom). Radix Tabs would model the tab+panel pair cleanly, but the picker also resets the search query on tab change (setSearch('')) and re-runs filterIcons. Wiring that through Radix Tabs' controlled API meant the same listener twice. The custom useState<Tab> + plain buttons keep the dataflow tight.
  • Token format. Tokens are "<set>:<name>" for SVG sets (e.g. vsc:VscFile, brand:react, ui:layers) and "emoji:<glyph>" for emojis. project-card's resolveIconToken() resolves both back to render output.
  • Search reset on tab change. Clicking a tab also runs setSearch(''), so the user always starts fresh when switching domains (icons in code and brands rarely share names).
  • ARIA listbox. <div role="listbox"> wraps the grid; each cell is role="option" with aria-selected. The search input has its own aria-label="Search icons".
  • No keyboard arrow navigation. Cells are plain buttons reachable by Tab. The grid does not implement ↑↓←→ roving tab-index — a follow-up if user testing flags it; tracked in docs/future-features.md.
  • Footer token strip. When value is set, a 24 px footer surfaces the canonical token in mono — useful when wiring the picker into a screenshot regression test that needs to assert which token was chosen.

Behavior
  • Controlled by consumer. Consumer holds value; the picker emits onSelect(token) on every cell click.
  • Search-within-tab. The search filter runs against the active tab's index only — switching tabs resets it.
  • Tab order. Code → Brands → UI → Emoji. Tabs are individual buttons (not a single roving listbox) so each tab is tabbable.
  • Selection survives filtering. If the user types something that excludes the selected cell, the selection token is preserved in value — but the cell visual is no longer visible. Clearing the search brings it back.
  • Icon rendering. Code, Brands, and UI sets are backed by react-icons component references stored in ./icons.ts (react-icons/vsc, react-icons/si, react-icons/lu respectively). Each IconEntry carries an Icon component; cells call <icon.Icon size={14} aria-hidden="true" /> directly — no dangerouslySetInnerHTML. Emoji glyphs render as plain text inside .emoji.
  • Token format. Stable, durable strings — vsc:VscFile, brand:react, ui:layers, emoji:🎨. Consumers must not parse the string; they pass it back to resolveIconToken() when rendering.
  • Embedded in dialogs. The picker is meant to sit inside a glass surface like new-project-dialog; on its own it has no scrim, no escape handler, no portal.
  • Reduced motion. Cell transitions are 100 ms each by spec; the token override doesn't apply (the values are hard-coded). This is a documented exception — the transitions are too tight to feel motion-aggressive even at reduce-motion preferences.
