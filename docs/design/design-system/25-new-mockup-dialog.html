New mockup dialog

Modal dialog that opens after the user drops a file on the app or clicks "New mockup" / "Upload mockup". Single recipe: nome+pickers+preview layout, tree-popover folder picker, color-coded file icon, inline-progress while uploading. Uses showCloseButton on RadixDialog.Content for the close-X — the close button is disabled (closeButtonDisabled) while uploading. Composes the canonical input-field for the name (driving field-level errors) and the alert-banner for non-field errors (server reject, network failure). The "replace as new version" toggle appears only when the dialog is opened from a mockup view.

React API — composes Radix Dialog + Radix Form + DS 27 + DS 28

The dialog is rendered with @radix-ui/react-dialog (modal + focus trap + escape + portal). The form inside uses @radix-ui/react-form for sync validation. The name field uses our input-field; the folder picker is a custom popover; global errors render via our alert-banner.

// NewMockupDialog.tsx — uses RadixDialog (DS 14) with showCloseButton
import { RadixDialog } from '@/components/Dialog/RadixDialog';   // see 14-dialog.html
import * as Form from '@radix-ui/react-form';
import { InputField, AlertBanner, FolderPicker } from '@/components/ds';

<RadixDialog.Root open={open} onOpenChange={setOpen}>
  <RadixDialog.Portal>
    <RadixDialog.Overlay />
    <RadixDialog.Content
      showCloseButton                              // CUSTOM — see DS 14 § close-btn
      closeButtonDisabled={isUploading}            // gray X while XHR in flight
    >

      <RadixDialog.Title>New mockup</RadixDialog.Title>

      {globalError && (
        <AlertBanner.Root status="error">                // DS 27 — non-field errors
          <AlertBanner.Title>{globalError.title}</AlertBanner.Title>
          <AlertBanner.Description>{globalError.detail}</AlertBanner.Description>
          <AlertBanner.Close asChild>
            <button onClick={() => setGlobalError(null)}><XIcon /></button>
          </AlertBanner.Close>
        </AlertBanner.Root>
      )}

      <FileChip file={file} />                            // HTML/ZIP color-coded

      <Form.Root onSubmit={handleSubmit}>

        <InputField.Root name="name"                    // DS 28 — field-level error/success
                          data-state={nameStatus}>     // 'error' | 'success' | undef
          <InputField.Label>Name</InputField.Label>
          <InputField.Control asChild>
            <input required pattern="[a-z0-9-]+" defaultValue={suggestedName} />
          </InputField.Control>
          <InputField.Message match="patternMismatch">
            Use lowercase letters, numbers, or hyphens only.
          </InputField.Message>
        </InputField.Root>

        <Form.Field name="projectId">
          <Form.Label>Project</Form.Label>
          <Form.Control asChild><select>{projectOptions}</select></Form.Control>
        </Form.Field>

        <FolderPicker                                  // custom — tree-popover variant
          projectId={selectedProject}
          value={folderPath}
          onChange={setFolderPath}
        />

        {currentViewIsMockup
          ? <ReplaceToggle currentMockup={currentMockup} />
          : <PreviewBox                                // renders 3 states via data-state:
              file={file}                              // loading | fallback | ready
              previewDataUrl={preview.dataUrl}         // from useFilePreview() (html2canvas)
              isLoading={preview.loading || uploading} // skeleton during gen AND upload
              statusLabel={
                uploading ? `uploading ${Math.round(progress*100)}%` :
                preview.loading ? 'generating…' :
                file.type === 'application/zip' ? 'not available for ZIP' : null
              }
            />}

        {uploading && <ProgressBar value={progress} />}    // determinate; XHR.upload.onprogress

        <Form.Submit asChild>
          <button className="btn-accent" disabled={uploading || nameStatus === 'error'}>
            {uploading ? 'Uploading…' : (globalError ? 'Retry' : 'Add mockup')}
          </button>
        </Form.Submit>
      </Form.Root>
    </RadixDialog.Content>
  </RadixDialog.Portal>
</RadixDialog.Root>
  • Radix Dialog handles: focus trap, Escape to close, scroll lock, portal, ARIA (aria-modal, labelledby, describedby).
  • Radix Form handles: sync validation via browser ConstraintValidation, emits [data-invalid] / [data-valid] on Field, Form.Message match="patternMismatch" binds error copy to the matcher.
  • InputField (DS 28) wraps Radix Form.Field with our styling. CSS picks up both Radix's [data-invalid] and our explicit [data-state="error"] (server-side name collision).
  • AlertBanner (DS 27) renders ONLY non-field errors at the top of the dialog body. Field-level errors stay on the input.
  • FolderPicker (custom compound, not yet a DS file) renders the tree-popover variant — chevron trigger + popover with project's folder tree.
  • FileChip + ReplaceToggle + PreviewBox are internal sub-components — not separate DS files since they only appear inside this dialog.
  • Submit button label is contextual: "Add mockup" (idle) / "Uploading…" (uploading) / "Retry" (after global error).

Behavior
  • Open paths: (a) file dropped on app via drop-overlay; (b) "New mockup" footer button (sidebar) → file picker → dialog; (c) empty-state dropzone click; (d) Ctrl/Cmd+U shortcut.
  • Name pre-fill: file basename minus extension, lower-cased, hyphenated. User can edit.
  • Project pre-fill: from route (current project / "Unsorted" for /). User can change.
  • Folder pre-fill: from route. User can navigate the tree-popover to a different folder.
  • Preview: file URL (URL.createObjectURL) rendered into an offscreen iframe + html2canvas screenshot → preview-box. While generating, preview-box shows [data-state="loading"] skeleton (shimmer gradient + 5 row blocks). On html2canvas timeout (3s) or error, falls back to [data-state="fallback"] with file icon. ZIP files skip the screenshot path and go straight to fallback. During upload, preview reverts to [data-state="loading"] skeleton (with label "uploading X%") and the determinate progress bar appears below — the preview area stays visible for size stability and signals "this view will refresh after upload". Replace-mode replaces the preview area with the replace-toggle radio.
  • Validation: name pattern ^[a-z0-9-]+$ (lowercase + digits + hyphen). Sync via Radix Form. Duplicate names checked async after debounce (200 ms) → field-error.
  • Submit: posts FormData to /api/mockups with file + name + projectId + folderId + (mode: add|replace). On success: dialog closes, navigates to the new mockup's view.
  • Cancel: aborts the request (AbortController), closes dialog. Esc closes the dialog when not uploading; when uploading, Esc is intercepted and shows a confirm via ConfirmDialog ("Cancel upload?").
  • Field error recovery: typing in the field clears its error state; submit re-runs validation.
  • Global error recovery: dismissing the banner clears it; Add button reverts to "Add mockup". Banner stays sticky otherwise (no auto-dismiss for errors).
  • Replace toggle behavior: when toggled to "Replace", the name field becomes read-only (matches the existing mockup's name) and the form submits to /api/mockups/[id]/version instead of /api/mockups.
