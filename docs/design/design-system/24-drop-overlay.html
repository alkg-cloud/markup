React API — custom compound (no Radix primitive)

Radix Primitives doesn't ship a drag-and-drop overlay — drag events are DOM-native and don't need primitive abstractions. The implementation is a custom compound rendered via @radix-ui/react-portal so it escapes any overflow:hidden ancestor.

// DropOverlay.tsx — portal-mounted custom compound
import { Portal } from '@radix-ui/react-portal';
import { useDragTarget } from '@/hooks/useDragTarget';

function AppDropOverlay() {
  const { isOver, target } = useDragTarget();    // listens to dragenter/leave on document
  if (!isOver) return null;

  return (
    <Portal>
      <DropOverlay.Root role="status" aria-live="polite">
        <DropOverlay.Panel>
          <DropOverlay.Icon />
          <DropOverlay.Title>Drop your HTML here</DropOverlay.Title>
          <DropOverlay.Path project={target.project} folder={target.folder} />
          <DropOverlay.Disclaimer>
            You can change the project and folder before submitting.
          </DropOverlay.Disclaimer>
        </DropOverlay.Panel>
      </DropOverlay.Root>
    </Portal>
  );
}
  • Drag detection is DOM-native (custom). useDragTarget listens to dragenter / dragleave at document level and resolves the current target from the route. Debounces leave by ~60 ms to survive transient leaves across child boundaries.
  • Portal-mounted via Radix Portal — overlay sits at body level, can't be clipped.
  • Aria role=status + aria-live=polite — announces target path without interrupting focus.
  • No close button — overlay unmounts on dragleave debounce or on drop. On drop, parent unmounts overlay and mounts NewMockupDialog pre-populated with the file + target.
  • Subcomponents accept asChild via @radix-ui/react-slot for icon / title customisation.

Behavior
  • Mounts on dragenter when the dragged data contains files (e.dataTransfer.types.includes('Files')).
  • Path preview reflects the current route. Routes: / → "Unsorted"; /projects/<slug> → project; /projects/<slug>/<folder…> → project + folder segments; mockup view inherits the parent folder's path.
  • Unmounts on debounced dragleave (60 ms) or on drop. The debounce survives transient leaves caused by child boundaries.
  • On drop: file read client-side, overlay unmounts, dialog mounts pre-populated with the file + target path.
  • Rejects (non-HTML/ZIP, multiple files, oversize) happen at the dialog, not here — the overlay always shows so the user confirms the drag was detected.
  • Iframe-rendered mockups (mockup-view): drag events on the iframe bubble to the outer document, so the overlay still shows during a drag over the mockup canvas.
