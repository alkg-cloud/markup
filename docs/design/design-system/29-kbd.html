Kbd

OS-aware keyboard-shortcut indicator. Renders each key as its own isolated keycap chip (<kbd> for semantics). macOS uses ⌘⇧⌥⌃ symbols with naked adjacency (no + between caps); Windows/Linux uses Ctrl/Shift/Alt text labels with an explicit + separator. Used in the topbar search pill (03 — Finder pill), the annotations-rail add button (18 — Annotations rail), and the command palette footer (04 — Finder overlay). No Radix primitive backs this component — custom at src/components/Kbd/.

React API

No Radix primitive backs this component — Radix Primitives ships no kbd or shortcut primitive. Custom compound at src/components/Kbd/Kbd.tsx. 

// Primary API — OS-aware combo
import { Kbd } from '@/components/Kbd/Kbd';

// mac:  [⌘] [⇧] [N]  — naked adjacency, no + between chips
// win:  [Ctrl] + [Shift] + [N]
<Kbd keys={['mod', 'shift', 'n']} />

// Single key
<Kbd keys={['esc']} />

// Disabled (CUSTOM — no Radix equivalent)
<Kbd keys={['mod', 'k']} disabled />

// Escape hatch — multi-glyph single chip (e.g. ↑↓ navigates up/down)
<Kbd.Group aria-label="up/down arrows">
  <Kbd.Key>↑↓</Kbd.Key>
</Kbd.Group>

// className pass-through for parent spacing
<Kbd keys={['mod', 'k']} className="my-offset" />
  • keys — ordered array of KbdKey tokens from keys.ts. Named modifiers: mod, shift, alt, ctrl. Named keys: enter, esc, tab, backspace, up, down, left, right. Any other string renders uppercase verbatim.
  • disabled — CUSTOM (no Radix primitive). Sets data-state="disabled" on the group wrapper → opacity 0.5 + cursor not-allowed. No consumer yet; CSS hook present for future use. Flagged at three places: state matrix row + this bullet + CSS comment.
  • Accessibility: each <kbd> is aria-hidden="true". The outer <span role="group"> carries an aria-label derived from announceCombo() (e.g. "shortcut: Command Shift N"). Screen readers announce the whole combo as one phrase.
  • Kbd.Group / Kbd.Key / Kbd.Plus — escape-hatch sub-components for non-standard chips. Callers provide their own aria-label on Kbd.Group.
  • Mac detection via isMac() in src/lib/shortcuts/platform.ts. Runs on every render (not cached as a constant) so SSR and hydration produce identical output when navigator is absent.
  • No + separator on mac — Apple HIG specifies naked adjacency for modifier combos.

Behavior
  • Not interactive — no hover, focus, or click states. Chip is purely presentational.
  • OS detection runs synchronously on each render via isMac() (reads navigator.platform / navigator.userAgent). Safe on the client (app is CSR-only); returns false when navigator is absent.
  • Mac: modifier keys resolved to glyphs (⌘ ⇧ ⌥ ⌃), naked adjacency (no "+" separator). Windows/Linux: text labels (Ctrl Shift Alt), "+" separator between chips.
  • Accessibility: individual <kbd> chips carry aria-hidden="true". The outer <span role="group"> provides a single aria-label (e.g. "shortcut: Command Shift N") so screen readers announce the combo as one phrase, not individual glyphs.
  • Disabled state: data-state="disabled" on the group dims the combo and sets cursor: not-allowed. No consumer uses this yet — the CSS hook exists for future use. Not announced to SR (chip is non-interactive).
  • Escape hatch: Kbd.Group + Kbd.Key allow multi-glyph single chips (e.g. ↑↓). Caller provides aria-label on Kbd.Group. Do not use for regular combos — use <Kbd keys={[…]} /> instead.
  • Plain-text shortcut for aria/tooltip use: formatShortcut() in src/lib/shortcuts/platform.ts (implicitly prepends the platform modifier). Use Kbd for visual chips; use formatShortcut() for aria-label + data-tooltip strings.
