08 — Dropdown Chevron

Animated disclosure chevron used by every expandable row in the project tree. Implemented in src/components/ProjectTree/TreeNode.tsx with the inline ChevronIcon from TreeIcons.tsx and CSS rules in ProjectTree.module.css. Right-pointing at rest; rotates 90° clockwise to point down when its parent row is expanded. Used by project rows, folder rows, and the recents group inside the sidebar tree. Leaf rows (mockups) substitute a .chevron-spacer so labels stay aligned.

Click a chevron or its parent row to toggle. Leaves (no children) render .chevron-spacer instead so labels stay aligned. Kebab fades in on row hover (see also sidebar).
React API — custom (no Radix primitive)

Radix UI's Primitives library does not currently ship a tree-view primitive (@radix-ui/react-collapsible exists but solves a different problem — single-section show/hide rather than a tree where one chevron per row drives both label-alignment and animated rotation). The project ships a custom compound in src/components/ProjectTree/ that owns expansion state via a Set<string> (useExpandedState.ts) and renders the chevron inline. The chevron itself is a native <button> with data-state driving rotation.

// src/components/ProjectTree/TreeNode.tsx — chevron rendering
import { ChevronIcon } from './TreeIcons';
import styles from './ProjectTree.module.css';

{node.expandable ? (
  <button
    type="button"
    tabIndex={-1}
    aria-hidden="true"                            // row owns aria-expanded
    className={cx(styles.chevron, node.expanded && styles.chevronOpen)}
    onClick={(e) => {
      e.stopPropagation();                          // don't fire row navigation
      onToggleExpand(node.id);
    }}
  >
    <ChevronIcon />
  </button>
) : (
  <span className={styles.chevronSpacer} />     // keeps leaf labels aligned
)}

// Parent row exposes the aria contract:
<div
  role="treeitem"
  aria-expanded={node.expandable ? node.expanded : undefined}
  aria-level={node.level}
  …
>

Notes:

  • The chevron <button> is aria-hidden and tabIndex=-1 — the parent div[role="treeitem"] owns the focus + aria-expanded contract per WAI-ARIA treeview pattern. Clicking the row also toggles expansion; the chevron button is a redundant hit-target convenience.
  • e.stopPropagation() on the chevron click is required: row onClick handles both toggle AND navigation (onNavigate(href)). Without it, clicking the chevron would also navigate.
  • Expansion state lives in the parent ProjectTree as a Set<string> (useExpandedState hook) keyed by stable node IDs — persists across re-renders and survives lazy-loaded children.
  • The ChevronIcon is inline SVG, not react-icons, so stroke weight matches the tree typography exactly (1.5 px stroke, 16 px viewBox).
State decision matrix
StateTriggerVisualAria
collapsed Default; expanded.has(id) === false. Chevron at 0°, color --text-muted. Children not in DOM. Parent row: aria-expanded="false".
expanded Click chevron or row; or keyboard Right-arrow. Chevron rotates 90° clockwise over --motion-fast. Color shifts to --accent. Children mount below. Parent row: aria-expanded="true".
leaf (custom) Node has no children (mockup row, recent-item). .chevron-spacer — 16 px reserved gutter, no glyph. Parent row omits aria-expanded entirely (not a disclosure).
focused Row keyboard-focused (chevron itself is tabIndex=-1). Standard --focus-ring on the row, not the chevron. Row has tabIndex={index === focusIndex ? 0 : -1} — roving tab.
"leaf" is a custom DS state with no Radix analogue — Collapsible is binary (open/closed) and has no notion of "non-collapsible peer slot to keep alignment". The spacer is essential when leaves and disclosable items share the same nesting level (folders vs mockups).
When to use which
PrimitiveUse forBacking
chevron (this) Tree-row disclosure — one chevron per expandable row, sharing a column with leaf-row spacers. Custom — ProjectTree compound.
@radix-ui/react-collapsible Single show/hide section ("Read more", FAQ accordion). Not installed in this project today. Radix Collapsible (would need to be added).

Rule of thumb: if the disclosed content is in-flow and the chevron must align with sibling-leaves' spacers, use this. If it's a floating panel, use popover.

Behavior
  • Toggle propagation. Clicking the chevron toggles expansion AND stops propagation, so it doesn't fire the row's onNavigate. Clicking the row body (anywhere outside the chevron and the kebab) both toggles expansion and navigates.
  • Roving tab. The chevron itself has tabIndex=-1; the row is the focusable target. Keyboard users use ArrowRight / ArrowLeft on the focused row to expand / collapse — matches the WAI-ARIA tree pattern.
  • Persistence. Expansion state is held by the parent ProjectTree in a stable Set<string>; it survives node re-orderings and lazy children loads.
  • Spacer. Leaf rows render .chevron-spacer (a 16 px non-interactive span), keeping their labels aligned with siblings' labels regardless of expansion state.
  • Color cue. Open chevrons re-tint to --accent so a glance distinguishes "open" from "closed" without inspecting rotation — useful at small font sizes.
  • Reduced motion. Under prefers-reduced-motion: reduce, the rotation collapses to 0 ms (token override at top of file) — the chevron snaps instead of sweeping.
