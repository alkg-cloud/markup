05 — Project Card

Tile that represents a project in the all-projects grid (/projects). Ships at src/components/ProjectCard/ProjectCard.tsx as a custom compound — no Radix primitive backs the card surface itself; the kebab menu rides on the project's usePopover hook (HTML native popover attribute, anchored via anchor-name). The card is one big next/link; the kebab is a sibling absolutely-positioned <button> so opening the menu never navigates. Glass treatment matches the projects-home tweaker exploration: accent-tinted fill (--accent-overlay-soft), --accent border, --shadow-glow halo, 14 px radius.

Auto-fill grid at minmax(280px, 1fr) with 16 px gap. Long names truncate via white-space:nowrap; overflow:hidden; text-overflow:ellipsis on both .card-name and .card-meta.
React API — custom compound + usePopover for the menu

No Radix primitive backs the card. The icon resolver (resolveIconToken) covers tokens emitted by the icon-picker (vsc:VscFile, brand:react, ui:layers, emoji:🎨); when the icon is null or points at an unknown token, a custom stroked window-tile glyph renders as the default. The kebab menu uses the project's usePopover hook (HTML native popover + anchor positioning) — the same hook that powers the sidebar tree kebab and the avatar menu.

import { ProjectCard, type ProjectCardData } from '@/components/ProjectCard/ProjectCard';

<ProjectCard
  project={{
    id: 'p_01',
    name: 'lumen-coffee',
    slug: 'lumen-coffee',
    icon: 'emoji:🎨',                              // from icon-picker
    mockupCount: 8,
    folderCount: 3,
  }}
  onOpen={() => router.push('/projects/lumen-coffee')}
  onEdit={() => setEditing(project)}
  onDelete={async () => (await confirm({ danger: true })) && deleteProject()}
/>

Notes:

  • Card is a next/link. The entire surface is one anchor — clicking anywhere except the kebab navigates to projectHref(slug). The kebab is a sibling absolutely-positioned <button> so its click doesn't bubble into the link.
  • Kebab via usePopover (custom). The hook returns triggerRef, triggerProps, popoverProps, and close(). It uses the HTML native popover attribute and CSS anchor-name for positioning. No Radix Dropdown/Menu primitive — the hook predates the Radix Dropdown adoption in this codebase and keeps the same surface for every kebab menu (sidebar, project card, folder card, avatar).
  • Icon resolution (custom). resolveIconToken(token) returns either an emoji or an SVG markup string. SVGs render via dangerouslySetInnerHTML (the picker only emits the project's pre-bundled vsc/brand/ui sets — no user-supplied SVG ever reaches this path). Unknown tokens fall back to DefaultProjectIcon.
  • Default glyph. A lightweight stroked window-tile shape (1.3 px rect + single header rule) — must stay in lockstep with the sidebar's default project glyph for visual coherence.
  • Pluralisation. pluralize(8, 'mockup', 'mockups') renders "8 mockups"; pluralize(1, ...) renders "1 mockup". Folders use the same helper.
  • Delete is destructive. onDelete is wired by the consumer to useConfirm({ danger: true }) — Radix AlertDialog enforces an explicit Cancel/Delete choice. See 14-dialog.

Behavior
  • Whole card navigates. The link wrapping the icon + info pair routes to projectHref(slug) via next/link. Prefetch on hover is automatic.
  • Kebab doesn't navigate. The kebab is a sibling button, not nested inside the link — clicking it opens the popover and leaves the route untouched.
  • Kebab reveal. Opacity 0 by default; 1 on card hover / focus-within / while its own menu is open. Pointer-events stay enabled so it's still keyboard-tabbable (focus-visible brings it back).
  • Menu actions. Three items — Open (router push), Edit (consumer-controlled, typically opens new-project-dialog in edit mode), Delete (consumer wires through useConfirm({ danger: true })).
  • Icon resolution fallback. Unknown or null tokens render the stroked window-tile glyph — the same default used by the sidebar tree.
  • Truncation. Long names ellipsis-truncate; the title attribute carries the full string for native browser tooltips.
  • Hover lift. Card translates −1 px on hover and back +1 px on active. Both collapse to none under prefers-reduced-motion: reduce.
  • Focus restoration. Closing the kebab menu (Escape / outside click) returns focus to the kebab button via usePopover's built-in restore logic.
