Empty state — drop-zone-gigante

Replaces the view content when a project / folder / all-projects view has zero mockups. Single recipe: a full-width dashed-bordered area that doubles as a drop target and a click-to-pick file affordance. Title + subtitle are context-aware (project name / folder name / "first mockup"). When the view has no mockups and the user has dragged a file, the drop-overlay takes over (this state hides behind it).

Default — on a project view
Dropzone fills the view width, 360px tall, dashed accent-overlay-mid border, glass-surface fill with 8px blur. Hover/focus brightens border to --accent and adds soft accent overlay.
Contextual titles per view
view: /
view: /projects/lumen-coffee
view: /projects/lumen-coffee/hero
Three view contexts share the same visual recipe; only title + subtitle copy change. The route resolves which copy to use.
React API — custom compound, uses native <input type="file">

Radix Primitives has no file-picker primitive — file selection is a browser-native concern. The implementation uses a native hidden <input type="file"> wrapped by a <label> so the entire dropzone is the click target (and Space/Enter activate the label). Drag events bind to the same wrapper.

// EmptyState.tsx — custom compound, native file input
function UploadEmptyState({ context }: { context: EmptyContext }) {
  const { title, sub, ariaLabel } = copyFor(context);    // route → copy lookup
  const handleFile = (file: File) => openNewMockupDialog({ file, target: context.target });

  return (
    <EmptyState.Root>
      <EmptyState.Dropzone
        aria-label={ariaLabel}
        onFile={handleFile}                            // debounced drag + native input change
      >
        <EmptyState.Icon />
        <EmptyState.Title>{title}</EmptyState.Title>
        <EmptyState.Subtitle>{sub}</EmptyState.Subtitle>
      </EmptyState.Dropzone>
    </EmptyState.Root>
  );
}
  • Native <input type="file" accept=".html,.zip"> hidden via clip:rect(0,0,0,0). The wrapping <label> makes the whole dropzone clickable AND keyboard-activatable (Enter / Space on the label).
  • Drag events (dragenter, dragover, drop) bind to the same <label>. The "Drag is happening over the whole app" overlay (drop-overlay) takes priority once it mounts.
  • No Radix primitive: file pickers are browser-native, and Radix deliberately stays out of the way.
  • Subcomponents (Icon, Title, Subtitle) accept asChild via @radix-ui/react-slot so consumers can swap any slot.
Behavior
  • Renders when the view has zero mockups AND the route is a project / folder / all-projects (mockup view never empties).
  • Click anywhere on the dropzone opens the system file picker (<label> activates the hidden input).
  • Keyboard: Tab to the dropzone, Enter / Space to open picker.
  • Drop a file onto the dropzone: same flow as drop on app surface — read file, mount new-mockup-dialog.
  • If the user starts a drag before the empty state mounts (mid-create), drop-overlay mounts on top and the empty state is occluded — they're functionally identical drop targets so this is fine.
  • Title copy: route lookup (project name, folder name, or "first mockup" for /).
  • Sub copy: "or click to choose a file" on / view; "Drop an HTML or click to upload to this [project|folder]" otherwise.
  • Rejects (non-HTML/ZIP, multiple files, oversize) propagate to the dialog as an inline alert-banner.
