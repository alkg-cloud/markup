09 — Avatar Menu

Top-right account menu. Implemented in src/components/Topbar/Topbar.tsx, opened via a circular .avatar-btn trigger and floated as a glass-surface popover. Personal items (Settings, Notifications) sit at the top group; admin-only items (Invites, Agent Tokens) live under an ADMIN subheading and are conditionally rendered with userRole === 'admin'. The destructive Sign Out entry uses the danger variant. Shares its glass recipe with dialog, popover, and tooltip.

React API — custom (no Radix dropdown-menu installed)

@radix-ui/react-dropdown-menu is the natural Radix backing for this surface, but it is not currently installed in this project's package.json (only react-dialog, react-alert-dialog, react-form, react-popover, react-portal, react-slot ship). The avatar menu is therefore implemented with the project's homegrown usePopover hook (which composes Radix Portal internally) and native <button role="menuitem"> children. Switching to @radix-ui/react-dropdown-menu is on the radar — see popover for the underlying mechanic.

// src/components/Topbar/Topbar.tsx — abridged
import { usePopover } from '@/lib/popover/usePopover';
import { VscKey, VscMail, VscSignOut } from 'react-icons/vsc';

const accountMenu = usePopover<HTMLButtonElement, HTMLDivElement>('right');

<button
  ref={accountMenu.triggerRef}
  type="button"
  className={styles.avatarBtn}
  aria-label="User menu"
  aria-haspopup="menu"                              // menu, not generic popover
  {...accountMenu.triggerProps}                       // onClick + aria-expanded
>{initial}</button>

<div {...accountMenu.popoverProps} className={styles.accountMenu} role="menu">
  <div className={styles.menuHeader}>
    <span className={styles.menuAvatar}>{initial}</span>
    <div>
      <span>{userName ?? 'User'}</span>
      {userEmail && <span>{userEmail}</span>}
    </div>
  </div>
  <div className={styles.menuDivider} />

  {userRole === 'admin' && (                       // admin-only block
    <>
      <div className={styles.menuSubhead}>Admin</div>
      <button role="menuitem" onClick={…}><VscMail /> Invites</button>
      <button role="menuitem" onClick={…}><VscKey /> Agent Tokens</button>
      <div className={styles.menuDivider} />
    </>
  )}

  <button role="menuitem" className={`${menuItem} ${menuItemDanger}`} onClick={handleLogout}>
    <VscSignOut /> Sign Out
  </button>
</div>

Notes:

  • usePopover handles outside-click dismissal, Escape-to-close, focus return, and portal mounting. It returns triggerProps (the onClick + aria-expanded) and popoverProps (positioning + visibility). The avatar menu adds role="menu" + aria-haspopup="menu" manually because usePopover is generic.
  • The command palette (open-command-palette event) and the avatar menu both occupy the top-layer; opening the palette closes the avatar menu via a document event listener so two overlays never stack.
  • The admin gate is purely cosmetic — server-side authorization still enforces access on /settings/invites and /settings/agents route handlers. Members navigating directly to those URLs are 403'd, not silently denied.
  • Future migration: when @radix-ui/react-dropdown-menu is added to deps, the structure maps cleanly: DropdownMenu.Root wraps; DropdownMenu.Trigger asChild takes the avatar button; DropdownMenu.Content replaces the popover div; DropdownMenu.Item replaces each <button role="menuitem">; DropdownMenu.Separator replaces .menuDivider; DropdownMenu.Label replaces .menuSubhead. Free additions: native arrow-key navigation, typeahead, RTL.
  • Native <button> elements with custom CSS classes — Radix has no Button primitive.
State decision matrix
StateTriggerVisualAria
closed Default. Trigger clicked to open. Popover absent from DOM (portal unmounted by usePopover). Trigger: aria-expanded="false".
open Click trigger, or Enter/Space while focused. Popover scales 0.96 → 1 + translateY −4 → 0 over --motion-base / --ease-spring. Trigger gains a 2 px --accent-soft ring. Trigger: aria-expanded="true". Popover: role="menu".
admin (custom) userRole === 'admin' from useIdentity(). Renders ADMIN subhead + Invites + Agent Tokens + divider. Hidden for members. Server-side auth still required on the destination routes.
item-hover Pointer over a .menu-item. --surface-hover bg, label brightens to --text-bright, icon inherits color. Native button focus semantics.
item-danger Item has [data-variant="danger"]. Resting label/icon in --danger; hover bg = --danger-soft. —
"admin" is a custom DS state — there's no Radix primitive for "conditional menu section by user role". Implemented as a JSX gate in the React tree.
When to use which
PrimitiveUse forBacking
avatar-menu (this) User-account dropdown anchored to the avatar trigger in the topbar. One per app shell. Custom: usePopover + role="menu".
popover Floating contextual UI without menu semantics — version pickers, info dropdowns. No role="menu". usePopover hook (composes Radix Portal).
dialog Modal forms or confirmations — focus-trap, scrim. Not for menu-of-actions. @radix-ui/react-dialog.
kebab menu Per-row contextual actions (rename, delete) in the project tree — see TreeNodeKebab. Same usePopover mechanic; different content.
Rule of thumb: dropdown-of-commands triggered by an avatar = this. Dropdown-of-commands triggered by a kebab on a row = TreeNodeKebab (same recipe, different content).

Behavior
  • Trigger click. Opens / closes the menu. Trigger gains aria-expanded="true" while open.
  • Escape closes. Returns focus to the trigger (handled by usePopover).
  • Outside click closes. Click anywhere outside the popover or trigger dismisses without action.
  • Item click runs action + closes. Each .menu-item handler calls accountMenu.close() before router.push(…) so the menu is gone by the time the new page mounts.
  • Sign Out. POSTs /api/auth/logout, awaits, then routes to /login. Closes the menu before the request fires.
  • Admin gate is cosmetic. Conditional render based on useIdentity().role; server-side handlers re-validate on /settings/*.
  • Conflict with command palette. Topbar listens for open-command-palette document events and closes itself when fired — palette and avatar menu never coexist.
  • Reduced motion. Enter animation collapses to a 0 ms opacity flip via the token override at the top of the file.
