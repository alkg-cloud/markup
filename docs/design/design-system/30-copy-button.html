CopyButton

Copy-to-clipboard button with inline feedback and screen-reader announcement via toast. Four visual variants share a single recipe: icon (default 28×28), ghost, secondary, and accent. Two custom states — copied and error — are not Radix primitives. The hook useCopy is also exported for non-button affordances (kebab-menu "Copy link", click-on-code).

React API — CopyButton + useCopy

Radix Primitives does NOT ship a Button or clipboard primitive. CopyButton is a custom compound at src/components/CopyButton/ that wraps a native <button> around the useCopy hook. The Toast channel provides the SR announcement. 

// Primary usage — icon-only (default variant)
import { CopyButton } from '@/components/CopyButton';

<CopyButton ariaLabel="Copy token" value={token} />

// Labelled secondary button in a reveal banner
<CopyButton
  variant="secondary"
  feedback="both"
  value={plaintextToken}
  ariaLabel={`Copy token ${name}`}
  message="Token copied — keep it safe"
>Copy</CopyButton>

// Hook only — for kebab-menu items, click-on-code, etc.
import { useCopy } from '@/components/CopyButton';

const { copy, copied } = useCopy({ message: 'Invite link copied' });
await copy(link); // returns true on success, false on failure
  • No Radix primitive backs this component — custom compound, no @radix-ui/react-button.
  • Icon-only rule: when children is absent, feedback is coerced to 'both' (even if 'inline' is passed) so SR users always receive a toast announcement — a silent icon swap is invisible to screen readers.
  • copied state (CUSTOM) — not a Radix primitive. Set for inlineDurationMs (default 1200 ms) after a successful clipboard write. Triggers the icon swap + accent tint + iconPop animation. Wired via data-state="copied" on the root button.
  • error state (CUSTOM) — not a Radix primitive. Set for inlineDurationMs when navigator.clipboard.writeText rejects (non-HTTPS or permission denied). Fires a "Copy failed" toast unconditionally. Wired via data-state="error" on the root button.
  • feedback controls channels: 'inline' (icon swap only), 'toast' (SR only), 'both' (default — both). Icon-only always overrides to 'both'.
  • Label text: when copied is true and children is present, the label renders "Copied!" instead of the original children text.

You need a standalone copy affordance (icon button, labelled button) The copy trigger is embedded in a non-button element (kebab-menu item, code block click, table cell)

Behavior
  • Click fires navigator.clipboard.writeText(value). On success: copied=true for inlineDurationMs, toast fires if feedback includes toast.
  • On failure (non-HTTPS / permission denied): error=true for inlineDurationMs, shake animation, "Copy failed" toast always fires.
  • Icon-only buttons always fire a toast on success regardless of the feedback prop — a silent icon swap is invisible to screen reader users.
  • The inline feedback duration is controlled by inlineDurationMs (default 1200 ms). The toast duration is controlled by toastDurationMs (default 3000 ms, matches DS 17 default).
  • copied and error are mutually exclusive. error overrides copied if a write fails immediately after a success (unlikely, but guarded).
  • Animations are state-attribute-driven — consumer sets data-state, CSS fires the keyframes. @media (prefers-reduced-motion: reduce) zeros all animation durations.
  • The component is a native <button type="button"> — keyboard-accessible without any ARIA additions. ariaLabel is required and passed as aria-label; no silent SR usage.
  • Custom icons override the copy (icon prop) and check (copiedIcon prop) slots. Both default to VSC codicons (VscCopy / VscCheck at 14 px).
