# HTML Mockup Review Platform — Product & Technical Specification

## Overview

A self-hosted web application for collaborative review and iteration of interactive HTML mockups generated by AI agents.

The platform acts as a lightweight “design review layer” on top of live HTML/CSS/JS mockups.

Unlike traditional design tools:

* mockups are real interactive HTML applications
* annotations are attached to screenshots of live states
* AI agents participate as reviewers/responders
* no visual editor exists
* the system focuses entirely on:

  * previewing
  * reviewing
  * annotating
  * iterating

Primary use case:

* integration with agent orchestrators like Paperclip
* AI-generated frontend iterations
* collaborative human + AI review workflows

---

# Goals

## Primary Goals

* Display interactive HTML mockups
* Allow screenshot-based annotations
* Preserve interactive behavior of mockups
* Enable AI agents to review/respond
* Keep architecture lightweight
* Run fully self-hosted in a single Docker container
* Integrate easily with Paperclip

---

# Non-Goals

The platform is NOT:

* a Figma replacement
* a drag-and-drop page builder
* a visual HTML editor
* a realtime multiplayer design system
* a CMS
* a deployment platform

# Core Concepts

## Mockup

A mockup is:

* an interactive HTML/CSS/JS application
* stored as static assets
* rendered inside sandboxed iframe

Each mockup contains:

* HTML
* CSS
* JS
* metadata
* comments
* screenshots
* annotation threads

---

## Annotation

An annotation is:

* a screenshot of a mockup state
* a drawing layer on top
* associated comments/thread

Annotations are immutable snapshots.

---

## Thread

A thread contains:

* user messages
* agent messages
* status
* resolution state

---

## Reviewer

Reviewer may be:

* human user
* AI agent

Both use the same thread model.

# Agent Interaction Model

Agents interact entirely through API.

Agents can:

* list mockups
* fetch annotations
* fetch screenshots
* fetch threads
* reply to threads
* mark resolved
* upload updated mockup builds

---

# Agent Review Flow

```text
1. Fetch annotation
2. Analyze screenshot + drawing JSON
3. Analyze thread
4. Modify HTML/CSS/JS
5. Upload new mockup version
6. Reply to thread
7. Mark resolved
```
